# 21. YOLO(You Only Look Once) — 주요 특징과 장점

본 글에서는 객체 탐지 분야의 패러다임을 바꾼 YOLO 모델이 무엇인지, 어떤 원리로 작동하는지, 그리고 어떤 강점과 한계를 가지는지를 하나의 흐름 속에서 정리합니다. 기존 2단계 탐지 방식과의 차이부터 버전별 발전 흐름까지, YOLO가 왜 객체 탐지의 표준 모델로 자리 잡았는지를 구조적으로 이해할 수 있도록 설명합니다.

## 1. YOLO란: 객체 탐지 문제를 다시 정의한 모델

딥러닝 기반 객체 탐지(object detection)는 이미지에서 객체의 위치(bounding box)와 클래스(class)를 동시에 찾아내는 문제입니다. YOLO 이전의 대표적인 접근 방식은 R-CNN 계열이었습니다. R-CNN은 먼저 객체가 있을 법한 후보 영역(region proposal)을 수천 개 생성한 뒤, 각 영역을 개별적으로 분류하는 2단계 방식이었습니다. 정확도는 높았지만 속도가 매우 느려 실시간 처리가 불가능했습니다.

YOLO(You Only Look Once)는 2015년 Joseph Redmon이 발표한 모델로, 이 패러다임을 완전히 바꿨습니다. 이름 그대로 이미지를 **단 한 번만 보고** 객체의 위치와 클래스를 동시에 예측합니다. 2단계 파이프라인을 1단계로 통합해 실시간 객체 탐지를 가능하게 만든 것이 YOLO의 핵심 기여입니다.

---

## 2. 핵심 아이디어: 탐지를 회귀 문제로 재정의

YOLO의 가장 혁신적인 점은 객체 탐지를 **회귀(regression) 문제**로 재정의했다는 것입니다.

기존 방식은 탐지를 분류 문제로 접근했습니다. 후보 영역을 먼저 찾고, 각 영역이 어떤 클래스인지 분류하는 두 단계였습니다. YOLO는 이 과정을 하나의 신경망이 한 번의 순전파로 처리합니다. 입력 이미지에서 바운딩 박스의 좌표, 크기, 신뢰도, 클래스 확률을 동시에 예측하는 단일 회귀 문제로 변환한 것입니다.

작동 방식은 다음과 같습니다. 입력 이미지를 S×S 그리드로 나눕니다. 각 그리드 셀은 B개의 바운딩 박스와 각 박스의 신뢰도(confidence score)를 예측합니다. 동시에 각 그리드 셀이 C개의 클래스 확률을 예측합니다. 이 모든 예측이 단 한 번의 순전파로 이루어집니다.

```
입력 이미지
    ↓
단일 CNN (한 번의 순전파)
    ↓
S×S 그리드 각 셀에서 동시에 예측:
├── 바운딩 박스 좌표 (x, y, w, h) × B개
├── 신뢰도 점수 (confidence) × B개
└── 클래스 확률 (C개 클래스)
    ↓
NMS(Non-Maximum Suppression)로 중복 제거
    ↓
최종 탐지 결과
```

## 3. 주요 특징

### 단일 신경망 구조 (Unified Architecture)

YOLO는 후보 영역 생성, 특성 추출, 분류를 별도의 컴포넌트로 나누지 않습니다. 하나의 신경망이 이미지 전체를 한 번에 처리해 탐지 결과를 직접 출력합니다. 이 단일 구조 덕분에 파이프라인 전체를 end-to-end로 학습할 수 있고, 각 컴포넌트를 별도로 최적화할 필요가 없습니다.

### 전체 이미지 컨텍스트 활용

YOLO는 이미지 전체를 한 번에 처리하므로, 각 그리드 셀이 이미지 전체의 맥락을 고려해 예측합니다. R-CNN 계열은 각 후보 영역을 독립적으로 처리하므로 주변 맥락 정보를 활용하기 어렵습니다. YOLO는 이미지 전체를 보기 때문에 배경을 객체로 잘못 탐지하는 오류(false positive)가 상대적으로 적습니다.

### 앵커 박스 (Anchor Box)

YOLOv2부터 도입된 개념으로, 데이터셋에서 자주 등장하는 바운딩 박스의 형태를 사전에 클러스터링해 앵커 박스로 정의합니다. 모델은 앵커 박스를 기준으로 오프셋(offset)을 예측하는 방식으로 바운딩 박스를 출력합니다. 처음부터 박스 좌표를 예측하는 것보다 학습이 안정적이고 정확도가 높아집니다.

### 다중 스케일 예측 (Multi-scale Prediction)

YOLOv3부터 도입된 기능으로, 서로 다른 크기의 특성 맵에서 탐지를 수행합니다. 큰 특성 맵에서는 작은 객체를, 작은 특성 맵에서는 큰 객체를 탐지합니다. Feature Pyramid Network(FPN)과 유사한 구조로, 다양한 크기의 객체를 더 잘 탐지할 수 있게 됩니다.

## 4. 장점

### 압도적인 속도: 실시간 처리 가능

YOLO의 가장 큰 강점은 속도입니다. 기존 R-CNN이 이미지 하나를 처리하는 데 수십 초가 걸렸다면, YOLOv1은 45FPS(초당 프레임)를 달성했습니다. 이후 버전들은 더욱 빨라져 YOLOv5, YOLOv8 같은 최신 버전은 GPU에서 수백 FPS를 달성합니다. 실시간 비디오 스트림 처리, 자율주행, 드론 탐지, 보안 카메라 분석처럼 속도가 핵심인 응용에서 YOLO가 선택되는 이유입니다.

### 높은 일반화 성능

YOLO는 이미지 전체를 한 번에 처리하기 때문에 객체의 전반적인 외형과 맥락을 학습합니다. 이 덕분에 다른 도메인의 이미지에 적용했을 때도 상대적으로 잘 작동합니다. 자연 이미지로 학습한 모델을 예술 작품이나 특수 촬영 이미지에 적용해도 어느 정도 탐지 성능을 유지합니다.

### End-to-End 학습

전체 탐지 파이프라인이 하나의 네트워크로 구성되므로, 손실 함수 하나로 전체 네트워크를 최적화할 수 있습니다. 별도로 학습된 컴포넌트들을 조합하는 방식보다 각 컴포넌트가 최종 탐지 결과에 맞게 통합적으로 최적화되어 더 효율적입니다.

### 지속적인 버전 발전

YOLO는 초기 버전 이후 커뮤니티와 연구자들에 의해 지속적으로 개선되어 왔습니다. YOLOv1(2015)에서 시작해 YOLOv8(2023), YOLOv9, YOLOv10까지 발전하면서 정확도와 속도가 꾸준히 향상되었습니다. 각 버전은 이전 버전의 한계를 극복하는 구조적 개선을 담고 있습니다.

## 5. 버전별 주요 개선 흐름

| 버전 | 출시 | 핵심 개선 |
|---|---|---|
| YOLOv1 | 2015 | 탐지를 회귀로 재정의, 실시간 처리 최초 달성 |
| YOLOv2 | 2016 | 앵커 박스 도입, 배치 정규화, 다중 스케일 학습 |
| YOLOv3 | 2018 | 다중 스케일 예측, Darknet-53 백본, 소형 객체 탐지 향상 |
| YOLOv4 | 2020 | CSPNet 백본, Mosaic 증강, 다양한 최적화 기법 통합 |
| YOLOv5 | 2020 | PyTorch 기반, 경량화, 배포 편의성 대폭 향상 |
| YOLOv8 | 2023 | Anchor-free 구조, 분할·포즈 추정 통합 지원 |

## 6. 한계

YOLO가 모든 상황에서 최선의 선택은 아닙니다. 몇 가지 구조적 한계가 있습니다.

그리드 셀 하나가 하나의 객체만 탐지할 수 있다는 제약이 있습니다. 초기 버전에서는 같은 그리드 셀 안에 여러 객체가 밀집해 있으면 탐지가 어려웠습니다. 이후 버전들에서 앵커 박스 수를 늘리고 다중 스케일 예측을 도입해 이 문제를 상당 부분 해결했습니다.

또한 매우 작은 객체 탐지에서는 2단계 방식 모델(Faster R-CNN 등)에 비해 정확도가 낮은 경우가 있습니다. 정밀한 탐지보다 속도가 더 중요한 상황에서 YOLO가 선택되는 이유가 여기에 있습니다.

| **주요 활용** | 자율주행, 보안 카메라, 드론, 실시간 비디오 분석 |

YOLO는 "정확도냐 속도냐"의 딜레마에서 속도를 선택한 모델이 아닙니다. 탐지 문제를 근본적으로 재정의해 두 가지를 동시에 달성하는 새로운 패러다임을 열었습니다. 이것이 YOLO가 10년 가까이 객체 탐지 분야의 표준 모델로 자리 잡고 있는 이유입니다.
