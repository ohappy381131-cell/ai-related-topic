# 14. 오토인코더(Autoencoder) — 적합한 상황과 인코더·디코더 구조

## 1. 오토인코더란: 스스로를 재현하는 신경망

오토인코더(Autoencoder)는 입력 데이터를 압축했다가 다시 복원하는 과정을 학습하는 비지도 학습 기반의 신경망입니다. 이름에서 드러나듯 스스로(auto) 인코딩하는 모델입니다.

얼핏 보면 이상한 목표처럼 느껴집니다. 입력을 받아서 그대로 출력하는 것을 학습한다면 그냥 입력을 복사하면 되지 않을까요? 오토인코더가 의미 있는 이유는 중간에 **병목(bottleneck)**이 있기 때문입니다. 입력을 그대로 통과시키지 않고, 의도적으로 훨씬 작은 차원의 잠재 공간(latent space)으로 압축했다가 다시 복원합니다. 이 과정에서 모델은 데이터의 핵심적인 구조와 패턴만을 잠재 공간에 담는 방법을 스스로 학습합니다.

```
입력 X → [인코더] → 잠재 벡터 z → [디코더] → 복원 X'

목표: X'이 X와 최대한 같아지도록 학습
손실: 재구성 오차(reconstruction error) = ||X - X'||²
```

정답 레이블이 없습니다. 입력 자체가 정답입니다. 이 점에서 오토인코더는 비지도 학습이면서 동시에 자기지도 학습(self-supervised learning)의 초기 형태로 볼 수 있습니다.

---

## 2. 오토인코더가 적합한 상황

오토인코더는 데이터의 잠재 구조를 학습한다는 특성 때문에 특정 문제 유형에 특히 강합니다.

### 차원 축소와 특성 학습

PCA가 선형 변환만 수행하는 것과 달리, 오토인코더는 비선형 활성화 함수를 사용하므로 데이터의 비선형 구조도 포착할 수 있습니다. 고차원 데이터를 저차원 잠재 벡터로 압축할 때, PCA로는 표현하기 어려운 복잡한 구조를 잠재 공간에 담을 수 있습니다. 이 잠재 벡터를 다른 머신러닝 모델의 입력 특성으로 사용하는 방식도 실무에서 자주 쓰입니다.

### 이상치 탐지 (Anomaly Detection)

오토인코더가 가장 강력하게 활용되는 영역 중 하나입니다. 핵심 아이디어는 이렇습니다. 정상 데이터로만 오토인코더를 학습시키면, 모델은 정상 데이터의 패턴을 잠재 공간에 담는 방법을 익힙니다. 이후 새로운 데이터를 복원했을 때 재구성 오차가 크면 그 데이터는 정상 패턴에서 벗어난 이상치일 가능성이 높습니다.

예를 들어 공장 설비의 정상 작동 데이터로 오토인코더를 학습시키면, 이후 고장 전 이상 진동 패턴은 복원이 잘 되지 않아 높은 재구성 오차를 보입니다. 신용카드 정상 거래 패턴을 학습한 오토인코더는 사기 거래를 높은 재구성 오차로 탐지할 수 있습니다. 레이블 없이 이상치를 탐지할 수 있다는 점이 결정적인 강점입니다.

### 노이즈 제거 (Denoising)

잡음 제거 오토인코더(Denoising Autoencoder)는 의도적으로 노이즈가 추가된 입력을 받아 원본 데이터를 복원하도록 학습합니다. 노이즈가 섞인 이미지를 입력하고 깨끗한 원본 이미지를 출력하도록 학습하면, 모델은 노이즈를 무시하고 본질적인 구조만 추출하는 방법을 익힙니다.

이미지 복원, 의료 영상 노이즈 제거, 오디오 잡음 제거 등에 활용됩니다. 노이즈가 추가된 입력도 결국 같은 잠재 공간에 매핑되어야 원본을 복원할 수 있으므로, 모델이 더 강건한 표현을 학습하게 되는 부수 효과도 있습니다.

### 데이터 생성 (Variational Autoencoder)

기본 오토인코더를 확률적으로 확장한 변분 오토인코더(VAE, Variational Autoencoder)는 잠재 공간을 연속적인 확률 분포로 모델링합니다. 잠재 공간에서 임의의 벡터를 샘플링해 디코더에 통과시키면 새로운 데이터를 생성할 수 있습니다. 이미지 생성, 텍스트 생성, 분자 구조 생성 등에 활용됩니다.

### 사전 학습 (Pre-training)

레이블 데이터가 부족한 상황에서, 레이블 없는 대량의 데이터로 오토인코더를 먼저 학습시켜 인코더가 유의미한 표현을 학습하게 한 뒤, 그 인코더를 지도 학습 모델의 초기값으로 사용합니다. 딥러닝 초기에는 이 방식이 깊은 네트워크 학습의 핵심 기법 중 하나였습니다.

## 3. 인코더 (Encoder)

인코더는 오토인코더의 전반부로, 고차원 입력 데이터를 저차원 잠재 벡터(latent vector)로 압축하는 역할을 합니다.

```
입력 X (고차원)
    ↓
[은닉층 1] → 차원 축소
    ↓
[은닉층 2] → 차원 축소
    ↓
잠재 벡터 z (저차원, 병목)
```

인코더는 입력에서 층을 거치면서 점진적으로 차원을 줄여나갑니다. 각 층에서 덜 중요한 정보는 버리고 핵심적인 구조만 남깁니다. 최종 출력인 잠재 벡터 z는 입력 데이터의 압축된 표현으로, 데이터의 본질적인 특성을 담은 코드(code)라고도 합니다.

이 잠재 벡터의 차원이 병목의 크기를 결정합니다. 차원이 작을수록 더 강하게 압축되어 모델이 더 핵심적인 정보만 선택적으로 보존해야 합니다. 반대로 너무 작으면 원본을 복원하기에 정보가 부족해집니다.

인코더 자체는 학습이 완료된 후 독립적으로 사용할 수 있습니다. 새로운 데이터를 인코더에 통과시켜 얻은 잠재 벡터를 다른 모델의 입력 특성으로 활용하는 방식이 대표적입니다.

---

## 4. 디코더 (Decoder)

디코더는 오토인코더의 후반부로, 인코더가 압축한 잠재 벡터를 다시 원본 데이터와 같은 차원으로 복원하는 역할을 합니다.

```
잠재 벡터 z (저차원)
    ↓
[은닉층 1] → 차원 확장
    ↓
[은닉층 2] → 차원 확장
    ↓
복원된 출력 X' (고차원, 입력과 같은 차원)
```

디코더는 인코더와 반대 방향으로 작동합니다. 잠재 벡터에서 시작해 층을 거치면서 점진적으로 차원을 늘려나가 원본 데이터의 차원을 복원합니다. 구조적으로도 인코더와 대칭 형태를 취하는 경우가 많습니다.

디코더의 목표는 잠재 벡터에 담긴 압축된 정보만으로 원본과 최대한 유사한 출력을 만드는 것입니다. 완벽한 복원은 불가능합니다. 압축 과정에서 일부 정보가 손실되기 때문입니다. 이 손실된 정보의 양이 재구성 오차로 측정됩니다.

데이터 생성 목적으로 사용할 때는 디코더를 독립적으로 활용합니다. VAE에서는 잠재 공간에서 무작위로 샘플링한 벡터를 디코더에 통과시켜 새로운 데이터를 생성합니다.

---

## 5. 인코더와 디코더의 비교

| | 인코더 (Encoder) | 디코더 (Decoder) |
|---|---|---|
| **역할** | 입력 데이터를 잠재 벡터로 압축 | 잠재 벡터를 원본 차원으로 복원 |
| **방향** | 고차원 → 저차원 | 저차원 → 고차원 |
| **구조** | 층을 거치며 차원 축소 | 층을 거치며 차원 확장 |
| **출력** | 잠재 벡터 z (병목) | 복원된 데이터 X' |
| **독립 활용** | 특성 추출기로 사용 | 데이터 생성기로 사용 |
| **핵심 질문** | "무엇이 핵심인가?" | "핵심에서 원본을 어떻게 재구성하는가?" |

두 구성 요소는 함께 학습되지만 역할이 정반대입니다. 인코더는 정보를 잃어가는 방향으로, 디코더는 잃어버린 정보를 추론해 채워넣는 방향으로 작동합니다. 학습이 완료된 후에는 목적에 따라 인코더만, 혹은 디코더만 독립적으로 활용할 수 있습니다.

---

## 6. 오토인코더의 전체 흐름

```
학습 단계:
입력 X → [인코더] → 잠재 벡터 z → [디코더] → X'
                                              ↓
                         재구성 오차 = ||X - X'||² 최소화

활용 단계:
이상치 탐지:  새 데이터 → [인코더+디코더] → 재구성 오차 크면 이상치
특성 추출:   새 데이터 → [인코더] → 잠재 벡터 → 다른 모델 입력
데이터 생성: 랜덤 벡터 → [디코더] → 새로운 데이터 (VAE)
노이즈 제거: 노이즈 데이터 → [인코더+디코더] → 복원된 깨끗한 데이터
```

---

## 핵심 요약

오토인코더는 입력을 압축했다가 복원하는 과정에서 데이터의 핵심 구조를 잠재 공간에 학습하는 비지도 신경망입니다. 이상치 탐지, 노이즈 제거, 차원 축소, 데이터 생성처럼 레이블 없이 데이터의 구조를 활용해야 하는 상황에 적합합니다. 인코더는 고차원 데이터를 저차원 잠재 벡터로 압축해 핵심 정보를 추출하고, 디코더는 잠재 벡터에서 원본 차원을 복원합니다. 두 구성 요소는 함께 학습되지만 역할이 정반대이며, 목적에 따라 독립적으로 활용할 수 있습니다.
