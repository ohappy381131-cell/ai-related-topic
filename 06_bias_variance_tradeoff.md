## 1. 모델이 틀리는 이유: 오차의 세 가지 원천

모델이 예측을 틀리는 이유는 무엇일까요? 단순히 "학습이 부족해서"라고 말하기에는 틀리는 방식이 서로 다릅니다. 어떤 모델은 항상 비슷한 방향으로 틀리고, 어떤 모델은 어떤 데이터를 주느냐에 따라 예측이 들쑥날쑥합니다. 이 두 가지 실패 방식은 원인이 다르고, 해결 방법도 다릅니다.

머신러닝에서 모델의 총 오차(error)는 수학적으로 세 가지 성분으로 분해됩니다.

```
총 오차 = 편향² + 분산 + 줄일 수 없는 노이즈(irreducible noise)
```

세 번째 항인 줄일 수 없는 노이즈는 데이터 자체에 내재된 불확실성으로, 어떤 모델을 써도 제거할 수 없습니다. 예를 들어 동일한 조건에서도 혈압이 조금씩 달라지는 것처럼, 세상에는 본질적으로 예측 불가능한 변동이 존재합니다. 우리가 실제로 통제하고 줄일 수 있는 것은 **편향과 분산**뿐입니다.

---

## 2. 편향 (Bias)

### 정의

편향은 모델의 예측값이 실제 정답에서 **체계적으로 얼마나 벗어나 있는가**를 나타냅니다. 핵심 단어는 "체계적"입니다. 무작위로 틀리는 것이 아니라, 항상 같은 방향으로, 비슷한 크기만큼 틀린다는 뜻입니다.

```
편향 = E[ŷ] - y
```

데이터를 바꿔가며 여러 번 학습해도 예측이 항상 실제값보다 높거나, 항상 낮거나, 항상 일정한 방향으로 빗나간다면 편향이 높은 것입니다.

### 원인: 과소적합 (Underfitting)

편향이 높은 근본 원인은 모델이 너무 단순하다는 것입니다. 데이터 안에 있는 복잡한 패턴을 모델의 표현력이 따라가지 못할 때 과소적합이 발생합니다. 비선형 관계가 있는 데이터에 단순한 선형 모델을 적용하는 것이 전형적인 예입니다. 실제 데이터가 곡선 형태(y = x² + noise)를 따르는데 모델은 직선(ŷ = ax + b)밖에 표현하지 못한다면, 아무리 오래 학습해도 직선이 곡선을 완벽히 따라갈 수 없습니다. 이 구조적 한계가 편향을 만들어냅니다.

### 과녁으로 이해하기

양궁 과녁으로 시각화하면 편향을 직관적으로 이해할 수 있습니다. 고편향 모델은 화살들이 서로 가까이 모여 있지만, 그 위치가 과녁 중심에서 멀리 벗어나 있습니다. 화살이 일관되게 왼쪽 아래에 꽂힌다면, 그 조준 자체에 체계적인 오류가 있는 것입니다. 데이터를 바꿔도 모델은 비슷한 영역에 예측을 내놓습니다.

### 고편향의 진단

고편향 모델의 가장 명확한 신호는 훈련 데이터에서의 성능도 낮다는 점입니다. 지도 학습에서 모델이 훈련 데이터조차 잘 맞추지 못한다면, 데이터를 더 많이 모아봐야 별 도움이 되지 않습니다. 모델 자체가 문제를 표현할 능력이 없기 때문입니다.

---

## 3. 분산 (Variance)

### 정의

분산은 훈련 데이터가 바뀔 때 모델의 예측이 **얼마나 크게 달라지는가**를 나타냅니다. 같은 문제를 풀기 위해 서로 다른 훈련 데이터로 모델을 여러 번 학습시켰을 때, 각 모델의 예측이 얼마나 들쑥날쑥한지를 측정합니다.

```
분산 = E[(ŷ - E[ŷ])²]
```

분산이 높다는 것은 모델이 훈련 데이터에 지나치게 민감하다는 뜻입니다. 어떤 데이터를 사용해서 학습하느냐에 따라 모델의 행동이 크게 달라집니다.

### 원인: 과적합 (Overfitting)

분산이 높은 근본 원인은 모델이 너무 복잡하다는 것입니다. 복잡한 모델은 훈련 데이터의 실제 패턴뿐만 아니라 노이즈까지 학습합니다. 예를 들어 10개의 데이터 포인트에 9차 다항식을 적용하면, 모델은 모든 점을 정확히 통과하는 곡선을 그릴 수 있습니다. 훈련 데이터에서의 오차는 거의 0이 됩니다. 그러나 이 모델은 훈련 데이터의 우연한 변동, 즉 노이즈까지 기억해버렸기 때문에, 새로운 데이터가 들어오면 예측이 심하게 벗어납니다. 패턴을 학습한 것이 아니라 데이터를 통째로 암기한 것입니다.

### 과녁으로 이해하기

고분산 모델은 화살들이 과녁 사방에 흩어져 있습니다. 특정 훈련 데이터로 학습한 모델은 중심 근처에 꽂히지만, 다른 훈련 데이터로 학습하면 완전히 다른 곳에 꽂힙니다. 어느 한 번의 예측이 우연히 정답과 가까울 수는 있지만, 일관성이 없어 신뢰할 수 없습니다.

### 고분산의 진단

고분산 모델의 가장 명확한 신호는 훈련 데이터에서의 성능은 높지만 테스트 데이터(혹은 검증 데이터)에서의 성능이 급격히 낮아진다는 점입니다. 두 성능 사이의 갭이 클수록 과적합이 심하다는 신호입니다.

## 4. 편향-분산 트레이드오프 (Bias-Variance Tradeoff)

편향과 분산은 독립적으로 동시에 낮출 수 없습니다. 편향을 줄이려고 모델을 복잡하게 만들면 분산이 늘어나고, 분산을 줄이려고 모델을 단순하게 만들면 편향이 늘어납니다. 이 역관계를 편향-분산 트레이드오프(Bias-Variance Tradeoff)라고 합니다.

```
오차
│
│╲                         고편향·저분산
│ ╲         총 오차         (너무 단순)
│  ╲       ╱─────╲
│   ╲     ╱       ╲──────  저편향·고분산
│    ╲   ╱  ← 최적점        (너무 복잡)
│편향²╲ ╱분산
│      ╳
│     ╱ ╲
│    ╱   ╲
└──────────────────────────── 모델 복잡도
   낮음   최적    높음
```

그래프를 보면 총 오차(편향² + 분산)는 U자형 곡선을 그립니다. 모델이 너무 단순할 때는 편향이 지배적이라 총 오차가 높고, 너무 복잡할 때는 분산이 지배적이라 총 오차가 다시 높아집니다. 그 사이 어딘가에 총 오차가 최소화되는 최적점이 존재합니다. 모델 설계의 핵심은 이 최적점을 찾는 것입니다.

| 모델 복잡도 | 편향 | 분산 | 현상 |
|---|---|---|---|
| 너무 낮음 | 높음 | 낮음 | 과소적합 (Underfitting) |
| 적절 | 낮음 | 낮음 | 최적 |
| 너무 높음 | 낮음 | 높음 | 과적합 (Overfitting) |

## 5. 진단 방법: 학습 곡선 (Learning Curve)

학습 곡선은 훈련 샘플 수를 점진적으로 늘려가면서 훈련 오차와 검증 오차가 어떻게 변하는지를 시각화한 그래프입니다. 모델이 과소적합인지 과적합인지를 진단하는 데 매우 유용한 도구입니다.

고편향(과소적합) 상황에서는 훈련 오차와 검증 오차 모두 높은 수준에서 수렴합니다. 둘 사이의 갭이 작다는 것은 훈련 데이터도 제대로 맞추지 못하고 있다는 뜻입니다. 데이터를 아무리 많이 추가해도 이 상황은 크게 개선되지 않습니다. 모델 자체의 표현력을 높여야 합니다.

고분산(과적합) 상황에서는 훈련 오차는 낮은데 검증 오차는 높습니다. 두 곡선 사이의 갭이 크다는 것이 핵심 신호입니다. 데이터를 더 많이 모으면 이 갭이 점차 줄어드는 경향이 있습니다. 또는 모델 복잡도를 줄이거나 규제를 적용해야 합니다.

---

## 6. 해결 전략

### 고편향(과소적합) 해결

고편향은 모델이 데이터의 패턴을 충분히 표현하지 못하는 문제이므로, 모델의 표현력을 높이는 방향으로 접근합니다. 더 깊은 신경망이나 고차 다항식처럼 모델 복잡도를 높이거나, 도메인 지식을 바탕으로 새로운 특성(feature)을 추가하는 피처 엔지니어링이 효과적입니다. 모델에 걸린 규제(regularization) 강도를 낮추거나, 학습을 더 오래 진행하는 것도 방법입니다.

### 고분산(과적합) 해결

고분산은 모델이 훈련 데이터의 노이즈까지 기억하는 문제이므로, 모델을 일반화하는 방향으로 접근합니다. 가장 직접적인 해결책은 훈련 데이터를 더 많이 확보하는 것입니다. 데이터가 늘어날수록 노이즈의 영향이 희석됩니다.

규제(Regularization)는 파라미터의 크기에 페널티를 부여해 모델이 지나치게 복잡해지지 않도록 억제합니다. L2 규제(Ridge)는 모든 파라미터를 작게 유지하고, L1 규제(Lasso)는 일부 파라미터를 아예 0으로 만들어 불필요한 특성을 제거합니다. 신경망에서는 드롭아웃(Dropout)을 적용해 학습 중 무작위로 일부 노드를 비활성화함으로써 특정 노드에 대한 의존도를 낮춥니다. 조기 종료(Early Stopping)는 검증 오차가 다시 상승하기 시작하는 시점에 학습을 중단해 과적합을 방지합니다.

| 전략 | 방법 | 대상 |
|---|---|---|
| 모델 복잡도 조절 | 레이어 수, 파라미터 수 변경 | 편향 또는 분산 |
| 데이터 추가 | 훈련 샘플 수 증가 | 분산 |
| 규제 (L1/L2) | 파라미터에 페널티 부여 | 분산 |
| 드롭아웃 | 랜덤 노드 비활성화 | 분산 |
| 조기 종료 | 검증 오차 상승 시 학습 중단 | 분산 |
| 피처 엔지니어링 | 새로운 특성 추가 | 편향 |

## 7. 앙상블이 분산을 줄이는 원리

앙상블(Ensemble) 방법은 여러 개의 모델을 결합해 단일 모델보다 성능을 높이는 기법입니다. 분산을 줄이는 데 특히 효과적인데, 그 이유를 수학적으로 이해하면 명확합니다.

서로 독립적인 n개의 모델이 각각 동일한 분산을 가진 예측을 내놓을 때, 이 예측들의 평균의 분산은 단일 모델 분산의 1/n이 됩니다. 모델이 많아질수록 각 모델이 가진 개별적인 오류가 평균 과정에서 상쇄되기 때문입니다. 이것이 Random Forest가 단일 Decision Tree보다 과적합에 훨씬 강한 이유입니다. 수백 개의 서로 다른 Decision Tree가 각자 조금씩 다른 예측을 내놓고, 그 평균이 어느 하나의 극단적 예측을 중화시킵니다.

그러나 앙상블이 분산만 줄이는 것은 아닙니다. Bagging과 Boosting은 서로 다른 목표를 가집니다. Bagging은 여러 모델을 병렬로 학습시키고 평균을 내어 주로 분산을 줄입니다. Boosting은 이전 모델이 틀린 데이터에 다음 모델이 집중하는 방식으로 순차적으로 학습해 주로 편향을 줄입니다. XGBoost와 LightGBM이 강력한 이유는 편향과 분산을 동시에 줄여나가는 Boosting 계열이면서, 과적합 방지 메커니즘도 내장하고 있기 때문입니다.

| 앙상블 방법 | 학습 방식 | 주로 줄이는 것 | 대표 알고리즘 |
|---|---|---|---|
| Bagging | 병렬 학습 후 평균 | 분산 | Random Forest |
| Boosting | 순차 학습, 오류 집중 | 편향 | XGBoost, LightGBM, AdaBoost |
| Stacking | 메타 모델로 결합 | 둘 다 | 다양한 조합 |

## 정리

| | 편향(Bias) | 분산(Variance) |
|---|---|---|
| **정의** | 예측의 체계적 오차 | 예측의 불안정성 |
| **원인** | 모델이 너무 단순 | 모델이 너무 복잡 |
| **현상** | 과소적합 (Underfitting) | 과적합 (Overfitting) |
| **진단** | 훈련·검증 오차 모두 높음 | 훈련↓ 검증↑ 갭 큼 |
| **해결** | 복잡도 증가, 특성 추가 | 데이터 추가, 규제, 드롭아웃 |

편향과 분산은 동시에 낮출 수 없습니다. 그렇다고 한쪽을 포기하라는 뜻도 아닙니다. 모델 설계의 핵심은 주어진 데이터와 문제의 성격을 정확히 진단한 뒤, 편향과 분산의 합이 최소가 되는 균형점을 체계적으로 찾아가는 과정입니다.
