# 1. 지도 학습과 비지도 학습 — 레이블의 존재 여부로 나뉘는 두 가지 학습 패러다임

본 글에서는 머신러닝의 두 핵심 학습 방식인 지도 학습(Supervised Learning)과 비지도 학습(Unsupervised Learning)의 개념, 구조, 알고리즘, 선택 기준을 하나의 흐름 속에서 정리합니다. 두 패러다임이 각각 어떤 문제를 풀기 위해 설계된 것인지, 그리고 실무에서 어떤 기준으로 선택하는지를 구조적으로 이해할 수 있도록 설명합니다.

## 1. 가장 근본적인 차이: 레이블(Label)

머신러닝 모델이 학습하는 방식은 결국 하나의 질문으로 귀결됩니다. **정답지가 있느냐, 없느냐.** 이 차이가 지도 학습과 비지도 학습을 구분하는 가장 본질적인 기준입니다.

지도 학습은 입력 데이터(X)와 그에 대응하는 정답 레이블(y)을 함께 제공합니다. 모델은 수많은 (X, y) 쌍을 반복적으로 보면서 X가 주어졌을 때 y를 정확히 예측하는 함수를 학습합니다. 마치 정답지가 딸린 문제집을 반복해서 풀며 패턴을 익히는 것과 같습니다.

비지도 학습은 반대로 입력 데이터(X)만 존재합니다. 모델에게 정답을 알려주지 않습니다. 대신 모델이 데이터 안에 숨어 있는 구조, 패턴, 군집을 스스로 발견하도록 설계됩니다. 정답이 없으므로 "맞다/틀리다"는 기준도 없습니다. 대신 "얼마나 의미 있는 구조를 찾아냈는가"로 평가합니다.

이 차이는 단순히 데이터 형태의 차이가 아닙니다. 두 패러다임은 근본적으로 다른 질문을 풀고 있습니다. 지도 학습은 "이것이 무엇인가?"를 묻고, 비지도 학습은 "이것들은 어떻게 연결되어 있는가?"를 묻습니다.

---

## 2. 지도 학습 (Supervised Learning)

### 작동 원리와 학습 과정

지도 학습의 흐름은 비교적 명확합니다. 먼저 레이블이 붙은 훈련 데이터를 준비합니다. 모델은 입력 X를 받아 예측값 ŷ을 출력하고, 이 예측값이 실제 정답 y와 얼마나 다른지를 손실 함수(loss function)로 계산합니다. 이 손실을 줄이는 방향으로 모델 내부의 파라미터를 조금씩 조정하는 과정을 수렴할 때까지 반복합니다. 충분히 학습된 모델은 이전에 본 적 없는 새로운 X가 들어와도 y를 높은 정확도로 예측할 수 있게 됩니다.

이 과정에서 핵심은 **일반화(generalization)**입니다. 훈련 데이터를 단순히 암기하는 것이 아니라, 데이터 안에 있는 보편적인 패턴을 학습해야 새로운 데이터에도 잘 동작합니다. 훈련 데이터에만 지나치게 맞춰진 모델은 실제 환경에서 성능이 급격히 떨어지는데, 이를 과적합(overfitting)이라고 합니다.

### 분류 (Classification)

분류는 입력 데이터를 미리 정의된 카테고리 중 하나로 분류하는 문제입니다. 출력이 연속적인 숫자가 아니라 이산적인 범주라는 점이 특징입니다. 이메일이 스팸인지 정상인지를 판단하거나, 종양 이미지를 악성과 양성으로 구분하거나, 고객이 이탈할지 유지될지를 예측하는 것이 모두 분류 문제입니다.

분류 문제에서는 정확도(Accuracy), 정밀도(Precision), 재현율(Recall), F1 점수 같은 지표로 모델 성능을 평가합니다. 클래스 간 데이터 수가 불균형할 때는 정확도만으로는 모델을 제대로 평가할 수 없어, 정밀도와 재현율의 균형을 함께 살펴야 합니다. 주로 사용되는 알고리즘으로는 Logistic Regression, Decision Tree, Random Forest, SVM, Neural Network 등이 있습니다.

### 회귀 (Regression)

회귀는 입력 데이터를 바탕으로 연속적인 수치를 예측하는 문제입니다. 집의 면적, 위치, 연식 같은 정보를 바탕으로 집값을 예측하거나, 광고 지출액을 바탕으로 매출액을 예측하는 것이 대표적인 예입니다. 출력이 숫자이므로 예측값과 실제값 사이의 거리를 오차로 계산하는 MSE(평균 제곱 오차), RMSE, MAE 같은 지표가 주로 사용됩니다. 주요 알고리즘으로는 Linear Regression, Ridge, Lasso, XGBoost 등이 있습니다.

### 지도 학습의 강점과 한계

지도 학습의 가장 큰 강점은 성능을 명확하게 측정할 수 있다는 점입니다. 정답이 있으므로 모델이 얼마나 잘하는지를 숫자로 비교할 수 있고, 이를 기반으로 모델을 개선하는 방향도 명확합니다. 실무 적용 범위도 넓어, 현재 산업에서 가장 많이 사용되는 방식입니다.

그러나 결정적인 한계가 있습니다. 레이블을 만드는 데 비용이 많이 든다는 것입니다. 의료 이미지에 전문의가 하나하나 레이블을 붙이거나, 텍스트 데이터에 감정 태그를 달아야 한다면 시간과 비용이 상당합니다. 또한 레이블 자체가 이미 인간의 판단과 편향을 담고 있기 때문에, 레이블에 오류나 편향이 있으면 그대로 모델에 학습됩니다.

## 3. 비지도 학습 (Unsupervised Learning)

### 작동 원리와 학습 과정

비지도 학습에서는 레이블 없이 입력 데이터만 모델에 제공합니다. 모델은 데이터 포인트들 사이의 거리, 밀도, 분산 같은 통계적 특성을 계산하면서 내부 구조를 탐색합니다. 그 결과로 군집, 저차원 표현, 이상치 같은 형태의 출력이 만들어집니다.

여기서 중요한 점은, 비지도 학습의 결과를 최종적으로 해석하는 것은 인간의 몫이라는 것입니다. 모델이 데이터를 세 개의 군집으로 나눠놓았다고 해도, 그 군집이 어떤 의미를 갖는지는 사람이 판단해야 합니다. 이 점이 지도 학습과 가장 결정적으로 다른 부분입니다.

### 군집화 (Clustering)

군집화는 비슷한 특성을 가진 데이터 포인트들을 자동으로 그룹화하는 방법입니다. 사람이 미리 카테고리를 정의하지 않아도, 알고리즘이 데이터 안에서 자연스럽게 형성된 그룹을 찾아냅니다. 예를 들어 수천만 명의 고객 구매 데이터를 군집화하면, 가격에 민감한 그룹, 브랜드 충성도가 높은 그룹, 계절에 따라 구매하는 그룹처럼 사전에 정의하지 않은 고객 세분화가 자동으로 도출됩니다.

K-Means는 가장 널리 쓰이는 군집화 알고리즘으로, 미리 군집의 수(K)를 지정하고 각 군집의 중심점을 반복적으로 업데이트하며 수렴합니다. DBSCAN은 데이터의 밀도를 기반으로 군집을 형성하므로, K를 미리 지정할 필요 없이 이상치(outlier)도 자동으로 탐지할 수 있습니다. Hierarchical Clustering은 데이터를 계층적 트리 구조로 묶어나가며, 군집의 수를 나중에 결정할 수 있다는 유연성이 있습니다.

### 차원 축소 (Dimensionality Reduction)

실제 데이터는 수백, 수천 개의 변수(차원)로 이루어진 경우가 많습니다. 차원이 높아질수록 데이터 간 거리가 의미를 잃고, 시각화도 불가능해지며, 계산 비용도 폭발적으로 늘어납니다. 이를 차원의 저주(curse of dimensionality)라고 합니다. 차원 축소는 이 문제를 해결하기 위해 고차원 데이터를 정보 손실을 최소화하면서 저차원으로 압축합니다.

PCA(주성분 분석)는 데이터의 분산이 가장 큰 방향으로 새로운 축을 설정해 차원을 줄이는 방법입니다. 수학적으로 명확하고 해석이 용이하지만, 선형 구조만 포착한다는 한계가 있습니다. t-SNE는 고차원에서의 데이터 간 유사도를 저차원에서도 최대한 보존하도록 설계된 비선형 방법으로, 2D나 3D 시각화에 특히 강합니다. UMAP은 t-SNE보다 계산이 빠르고 전역적인 구조도 더 잘 보존합니다.

### 이상치 탐지 (Anomaly Detection)

이상치 탐지는 대부분의 데이터가 보이는 정상적인 패턴을 먼저 학습한 뒤, 그 패턴에서 벗어나는 데이터를 이상치로 탐지하는 방법입니다. 신용카드 사기 거래 탐지, 제조 공정의 불량 탐지, 네트워크 침입 탐지 등이 대표적인 응용 사례입니다. Autoencoder, Isolation Forest, One-Class SVM 등이 주로 사용됩니다.

### 비지도 학습의 강점과 한계

비지도 학습의 가장 큰 강점은 레이블 없이 대량의 데이터를 바로 활용할 수 있다는 점입니다. 현실의 데이터 대부분은 레이블이 없으므로, 비지도 학습은 레이블링 비용 없이 데이터를 탐색하는 강력한 도구입니다. 특히 사전에 무엇을 찾을지 모를 때, 즉 탐색적 분석 단계에서 진가를 발휘합니다.

한계는 결과의 해석과 평가가 주관적이라는 점입니다. 지도 학습처럼 "정확도 92%"처럼 객관적인 수치로 성능을 비교하기 어렵고, 발견된 군집이나 패턴이 실제로 의미 있는지를 도메인 전문가가 직접 검토해야 합니다.

---

## 4. 두 패러다임 비교

| 항목 | 지도 학습 | 비지도 학습 |
|---|---|---|
| **레이블** | 필수 | 없음 |
| **목표** | 예측 | 구조 발견 |
| **성능 평가** | 정확도, F1, RMSE 등 수치 | 실루엣 계수, 시각적 해석 |
| **인간 역할** | 레이블 제공 | 결과 해석 |
| **대표 과제** | 스팸 분류, 가격 예측 | 고객 세분화, 이상치 탐지 |

---

## 5. 그 외 주요 패러다임

현업에서는 지도/비지도의 이분법이 자주 깨집니다. 소량의 레이블 데이터와 대량의 비레이블 데이터를 함께 활용하는 **준지도 학습(Semi-supervised Learning)**은 레이블링 비용을 줄이면서도 지도 학습의 성능을 일부 유지할 수 있습니다. **자기지도 학습(Self-supervised Learning)**은 데이터 자체에서 레이블을 자동으로 생성하는 방식으로, GPT나 BERT 같은 대형 언어 모델의 사전학습(pre-training)이 대표적인 예입니다. **강화 학습(Reinforcement Learning)**은 보상과 페널티 신호를 통해 환경과 상호작용하며 행동 정책을 최적화하는 방식으로, 정답이 아닌 결과의 좋고 나쁨을 기준으로 학습합니다.

---

## 6. 선택 기준 가이드

```
데이터에 레이블이 있는가?
├── YES → 지도 학습
│         출력이 범주형? → 분류 (Classification)
│         출력이 연속형? → 회귀 (Regression)
│
└── NO  → 목적이 무엇인가?
          패턴 발견/군집    → 군집화 (K-Means, DBSCAN)
          구조 압축/시각화  → 차원 축소 (PCA, UMAP)
          이상치 탐지       → 밀도 추정 (Autoencoder)
```

지도 학습은 인간이 이미 정의한 세계관을 모델에 이식하는 행위입니다. 비지도 학습은 데이터가 스스로 드러내는 구조를 포착하는 행위입니다. 두 접근은 서로 경쟁하는 것이 아니라, 문제의 성격과 데이터의 상태에 따라 선택하거나 결합하는 보완적 도구입니다.
