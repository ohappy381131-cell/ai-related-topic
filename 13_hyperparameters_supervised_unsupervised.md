# 지도 학습과 비지도 학습에서의 하이퍼파라미터 — 역할 비교

본 글에서는 하이퍼파라미터가 딥러닝에만 국한된 개념이 아님을 출발점으로, 지도 학습과 비지도 학습 각각에서 하이퍼파라미터가 어떤 역할을 하는지를 하나의 흐름 속에서 비교합니다. 두 패러다임이 풀려는 문제의 성격이 다르기 때문에, 하이퍼파라미터가 모델에 미치는 영향의 방식과 조정 기준도 어떻게 달라지는지를 구조적으로 이해할 수 있도록 설명합니다.

## 1. 하이퍼파라미터는 머신러닝 전반의 개념이다

하이퍼파라미터는 딥러닝에만 국한된 개념이 아닙니다. 머신러닝 전반, 즉 지도 학습과 비지도 학습 모두에서 등장합니다. 다만 두 패러다임이 풀려는 문제의 성격이 다르기 때문에, 하이퍼파라미터가 하는 역할과 그것이 모델에 미치는 영향의 방식도 달라집니다.

공통점은 하나입니다. 하이퍼파라미터는 학습이 시작되기 전에 사람이 직접 설정해야 하는 값이며, 모델이 데이터로부터 스스로 학습하지 못합니다. 그리고 어떻게 설정하느냐에 따라 같은 데이터, 같은 알고리즘을 사용해도 결과가 크게 달라집니다.

## 2. 지도 학습에서의 하이퍼파라미터

지도 학습은 입력(X)과 정답(y)이 있고, 모델이 X→y 매핑을 학습합니다. 이때 하이퍼파라미터는 주로 **모델이 얼마나 복잡하게 학습할지**, 그리고 **과적합을 얼마나 억제할지**를 결정합니다. 정답이 있으므로 하이퍼파라미터 조정의 기준도 명확합니다. 검증 데이터에서의 정확도, F1, RMSE 같은 수치가 하이퍼파라미터 선택의 직접적인 기준이 됩니다.

### 선형 모델 (Linear/Logistic Regression)

선형 모델에서 가장 중요한 하이퍼파라미터는 규제 강도(λ)입니다. 규제는 모델의 파라미터가 지나치게 커지는 것을 억제해 과적합을 방지합니다.

L1 규제(Lasso)는 일부 파라미터를 정확히 0으로 만들어 불필요한 특성을 자동으로 제거하는 효과가 있습니다. 변수 선택(feature selection)이 필요한 상황에서 유용합니다. L2 규제(Ridge)는 모든 파라미터를 작게 유지하되 0으로 만들지는 않습니다. ElasticNet은 L1과 L2를 결합한 방식으로, 두 규제의 비율(l1_ratio)도 하이퍼파라미터가 됩니다.

λ가 너무 크면 모델이 지나치게 단순해져 과소적합이 발생하고, 너무 작으면 규제 효과가 없어 과적합이 발생합니다.

### 결정 트리 (Decision Tree)

결정 트리의 하이퍼파라미터는 트리가 얼마나 깊이, 얼마나 복잡하게 성장할지를 제어합니다.

최대 깊이(max_depth)는 트리가 몇 단계까지 분기할 수 있는지를 제한합니다. 깊을수록 훈련 데이터를 세밀하게 학습하지만 과적합 위험이 커집니다. 분기 최소 샘플 수(min_samples_split)는 하나의 노드를 분기하려면 최소 몇 개의 샘플이 있어야 하는지를 결정합니다. 리프 노드 최소 샘플 수(min_samples_leaf)는 리프 노드에 최소 몇 개의 샘플이 있어야 하는지를 결정합니다. 이 두 값이 클수록 트리 성장이 억제되어 과적합이 줄어들지만 표현력도 낮아집니다.

### 앙상블 모델 (Random Forest, XGBoost, LightGBM)

앙상블 모델은 하이퍼파라미터의 수가 많고, 각각이 성능에 미치는 영향도 큽니다.

트리의 수(n_estimators)는 몇 개의 트리를 결합할지를 결정합니다. 많을수록 분산이 줄어들고 성능이 안정적으로 높아지지만, 어느 수준 이상에서는 계산 비용만 늘고 성능 향상은 미미해집니다. 학습률(learning_rate)은 부스팅 계열에서 각 트리의 기여도를 조절합니다. 낮을수록 과적합이 억제되지만 더 많은 트리가 필요합니다. 서브샘플 비율(subsample)은 각 트리를 학습할 때 사용할 훈련 데이터의 비율입니다. 1보다 작게 설정하면 무작위 샘플링으로 분산을 줄일 수 있습니다.

### SVM (Support Vector Machine)

SVM에서는 C와 커널(kernel) 관련 파라미터가 핵심입니다. C는 마진(margin)과 오분류 사이의 트레이드오프를 조절합니다. C가 크면 오분류를 최소화하려 해 복잡한 결정 경계를 만들고(과적합 위험), 작으면 넓은 마진을 선호해 더 단순한 경계를 만듭니다. RBF 커널을 사용할 때는 감마(gamma)도 중요합니다. gamma가 크면 각 훈련 샘플의 영향 범위가 좁아져 복잡한 경계가 형성됩니다.

### KNN (K-Nearest Neighbors)

KNN에서는 K, 즉 몇 개의 이웃을 참조할지가 핵심 하이퍼파라미터입니다. K가 작으면 노이즈에 민감하고 과적합 위험이 있으며, K가 크면 경계가 부드러워지지만 지역적 패턴을 놓칠 수 있습니다.

## 3. 비지도 학습에서의 하이퍼파라미터

비지도 학습은 정답 레이블이 없습니다. 모델이 데이터 내부의 구조를 스스로 발견하는 것이 목표입니다. 이 때문에 하이퍼파라미터의 역할이 지도 학습과 근본적으로 다릅니다. 지도 학습에서는 하이퍼파라미터가 **예측 정확도**를 최적화하는 방향으로 조정되지만, 비지도 학습에서는 **발견한 구조가 얼마나 의미 있는가**를 기준으로 조정됩니다. 그리고 이 기준 자체가 훨씬 주관적입니다.

### K-Means 군집화

K-Means에서 가장 중요한 하이퍼파라미터는 군집의 수 K입니다. 그런데 이것이 K-Means의 가장 어려운 문제이기도 합니다. 정답 레이블이 없으므로 "K=3이 맞다"고 검증할 기준이 없습니다.

K를 결정하는 데 주로 사용하는 방법이 엘보우 방법(Elbow Method)입니다. K를 1부터 늘려가면서 각 K에서의 군집 내 분산(inertia)을 그래프로 그리면, 어느 지점에서 감소 폭이 급격히 줄어드는 "팔꿈치(elbow)" 모양이 나타납니다. 이 지점을 최적 K로 선택합니다. 실루엣 계수(Silhouette Score)도 자주 사용됩니다. 각 샘플이 자신의 군집과 얼마나 가깝고 다른 군집과 얼마나 먼지를 -1~1 사이 값으로 나타내며, 높을수록 군집 구조가 명확합니다.

초기 중심점(init)도 하이퍼파라미터입니다. K-Means는 초기 중심점 설정에 따라 결과가 달라질 수 있어, k-means++ 방식으로 초기화하는 것이 일반적으로 더 안정적인 결과를 냅니다.

### DBSCAN

DBSCAN은 K를 미리 지정할 필요가 없는 대신 두 개의 하이퍼파라미터가 군집 구조 전체를 결정합니다. 입실론(eps)은 한 점의 이웃으로 간주할 최대 거리입니다. eps가 크면 더 많은 점이 이웃으로 포함되어 군집이 크게 형성되고, 작으면 군집이 작게 쪼개지거나 많은 점이 이상치로 분류됩니다. 최소 샘플 수(min_samples)는 핵심 포인트(core point)가 되기 위해 eps 반경 안에 있어야 하는 최소 이웃 수입니다. 이 값이 크면 밀도 기준이 높아져 이상치가 늘어납니다.

두 값의 조합이 군집의 수, 크기, 이상치 비율을 동시에 결정하기 때문에, DBSCAN의 하이퍼파라미터 설정은 도메인 지식이 중요합니다.

### PCA (주성분 분석)

PCA에서는 몇 개의 주성분을 유지할지, 즉 축소할 차원의 수(n_components)가 핵심 하이퍼파라미터입니다. 이를 결정하는 기준으로 **설명 분산 비율(explained variance ratio)**이 사용됩니다. 주성분을 하나씩 추가할 때마다 누적 설명 분산이 얼마나 늘어나는지를 확인하고, 일반적으로 누적 설명 분산이 85~95%가 되는 지점에서 차원 수를 결정합니다. 정보를 얼마나 보존할지와 차원을 얼마나 줄일지 사이의 트레이드오프입니다.

## 4. 지도 학습 vs 비지도 학습: 하이퍼파라미터 역할 비교

| 항목 | 지도 학습 | 비지도 학습 |
|---|---|---|
| **주요 역할** | 예측 정확도와 일반화 성능 최적화 | 데이터 구조 발견의 질 조절 |
| **조정 기준** | 검증 데이터 성능 (정확도, F1, RMSE) | 실루엣 계수, 설명 분산, 시각적 해석 |
| **평가 방식** | 객관적 수치로 비교 가능 | 주관적 해석 포함, 도메인 지식 필요 |
| **핵심 트레이드오프** | 편향-분산 (과소적합 vs 과적합) | 군집 수·밀도·차원 수의 의미 있는 선택 |
| **자동화 난이도** | 상대적으로 용이 (그리드/베이지안 탐색) | 상대적으로 어려움 (정답 없음) |
| **대표 하이퍼파라미터** | 규제 강도(λ), 트리 깊이, 학습률 | K, eps, n_components |

## 5. 공통 원칙: 어느 쪽이든 사람의 판단이 필요하다

지도 학습에서도, 비지도 학습에서도 하이퍼파라미터 설정에는 결국 사람의 판단이 개입합니다. 지도 학습에서는 검증 성능이라는 객관적 기준이 있어 자동화가 더 수월하지만, 어떤 평가지표를 최적화할지(정확도인지 재현율인지)는 비즈니스 목표에 따라 사람이 결정해야 합니다. 비지도 학습에서는 발견한 군집이나 차원 구조가 실제로 의미 있는지를 도메인 전문가가 검토해야 합니다.

공통적으로 적용할 수 있는 원칙은 이렇습니다. 단순한 모델에서 시작해 점진적으로 복잡도를 높이고, 각 단계에서 하이퍼파라미터 변화가 결과에 어떤 영향을 미치는지를 기록하며, 과적합과 과소적합 사이의 균형을 지속적으로 확인합니다.

---

## 핵심 요약

하이퍼파라미터는 딥러닝만의 개념이 아니라 머신러닝 전반에 걸친 개념입니다. 지도 학습에서는 정답이 있으므로 하이퍼파라미터 조정의 목표와 기준이 명확합니다. 모델의 복잡도와 규제 강도를 조절해 예측 성능과 일반화 능력의 균형을 찾습니다. 비지도 학습에서는 정답이 없으므로 하이퍼파라미터가 발견할 구조의 형태 자체를 결정하며, 그 결과가 의미 있는지를 판단하는 데 도메인 지식과 주관적 해석이 필요합니다.
