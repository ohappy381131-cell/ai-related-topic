## 1. 전이 학습이란

딥러닝 모델을 처음부터 학습시키려면 방대한 양의 데이터와 막대한 계산 자원이 필요합니다. ImageNet처럼 1,000개 클래스, 1,200만 장의 이미지로 ResNet을 학습시키는 데는 고성능 GPU 클러스터에서도 수일이 걸립니다. 대부분의 실무 환경에서는 이런 자원을 확보하기 어렵습니다.

전이 학습(Transfer Learning)은 이 문제를 우회하는 방법입니다. 대규모 데이터셋으로 이미 학습된 모델의 지식을 새로운 문제에 재사용하는 접근입니다. 사람이 자전거 타는 법을 배운 뒤 오토바이를 훨씬 빠르게 배우는 것처럼, 한 도메인에서 학습한 지식이 다른 도메인 학습을 가속합니다.

핵심 전제는 **저수준 특성은 도메인을 초월해 공유된다**는 것입니다. 이미지 인식에서 선, 모서리, 질감, 형태 같은 저수준 특성은 고양이를 인식하든 자동차를 인식하든 공통적으로 유용합니다. ImageNet으로 학습된 모델의 초기 레이어들은 이미 이런 범용적인 시각 특성 추출기로 기능합니다. 이것을 새로운 문제에 그대로 재활용하는 것이 전이 학습의 본질입니다.

---

## 2. 왜 전이 학습이 효과적인가

전이 학습이 단순히 편의를 위한 기법이 아니라 실질적으로 효과적인 이유를 이해하려면, 딥러닝 모델이 각 레이어에서 무엇을 학습하는지를 살펴야 합니다.

CNN의 레이어를 시각화한 연구들에 따르면, 학습된 모델의 레이어는 계층적 구조를 가집니다. 초기 레이어는 선, 모서리, 색상 경계 같은 매우 일반적인 저수준 특성을 탐지합니다. 이 특성들은 어떤 이미지 데이터셋에서 학습하든 비슷하게 나타납니다. 중간 레이어는 질감, 패턴, 부분적 형태 같은 중간 수준의 특성을 탐지합니다. 후반 레이어는 특정 도메인에 특화된 고수준 특성을 탐지합니다. 얼굴 인식 모델이라면 눈, 코, 입 전체의 구조를 탐지합니다.

전이 학습은 이 계층 구조를 활용합니다. 도메인이 달라도 저수준과 중간 수준의 특성 추출 능력은 재사용할 수 있고, 후반 레이어만 새로운 도메인에 맞게 조정하면 됩니다.

---

## 3. 사전 학습 모델 (Pre-trained Model)

전이 학습의 출발점은 사전 학습 모델(pre-trained model)입니다. 대규모 데이터셋으로 이미 학습이 완료된 모델을 말합니다. 이미지 분류 분야에서는 주로 ImageNet으로 학습된 모델들이 사용됩니다.

대표적인 사전 학습 모델들은 다음과 같습니다. VGG16, VGG19는 단순하고 직관적인 구조로 전이 학습의 베이스라인으로 자주 사용됩니다. ResNet-50, ResNet-101은 잔차 연결(residual connection) 구조로 깊은 네트워크 학습을 가능하게 한 모델로, 전이 학습에서 가장 널리 사용됩니다. EfficientNet은 정확도와 계산 효율의 균형이 뛰어나 현재 가장 인기 있는 사전 학습 모델 중 하나입니다. Vision Transformer(ViT)는 Transformer 구조를 이미지에 적용한 모델로, 대규모 데이터에서 뛰어난 성능을 보입니다.

PyTorch에서는 `torchvision.models`, Hugging Face에서는 `transformers` 라이브러리를 통해 이 모델들을 몇 줄의 코드로 불러올 수 있습니다.

```python
import torchvision.models as models

# ImageNet으로 사전 학습된 ResNet-50 불러오기
model = models.resnet50(pretrained=True)
```

## 4. 전이 학습의 두 가지 전략

이미지 분류 문제에서 전이 학습을 적용하는 방식은 크게 두 가지입니다. 어떤 전략을 선택할지는 새로운 데이터셋의 크기와 사전 학습 도메인과의 유사성에 따라 달라집니다.

### 특성 추출 (Feature Extraction)

사전 학습 모델의 가중치를 고정(freeze)하고, 마지막 분류 레이어만 새로운 클래스에 맞게 교체해 학습하는 방법입니다.

```
사전 학습 모델
├── 합성곱 레이어들 (가중치 고정, 학습 안 함)
│    레이어 1: 저수준 특성 추출
│    레이어 2: 중간 수준 특성 추출
│    ...
│    레이어 N: 고수준 특성 추출
└── 분류 레이어 (교체 후 학습)
     기존: 1000개 클래스 (ImageNet)
     새로: N개 클래스 (새 문제)
```

합성곱 레이어들은 특성 추출기(feature extractor)로만 작동합니다. 새로운 이미지를 입력하면 고정된 가중치가 특성 벡터를 만들어내고, 이 벡터를 새로운 분류 레이어가 학습합니다. 이 방법은 새로운 데이터가 적거나 사전 학습 도메인과 유사할 때 적합합니다. 학습해야 할 파라미터가 적으므로 빠르게 수렴하고 과적합 위험도 낮습니다.

```python
# ResNet-50의 모든 파라미터 고정
for param in model.parameters():
    param.requires_grad = False

# 마지막 분류 레이어만 교체 (1000 → 10 클래스)
import torch.nn as nn
model.fc = nn.Linear(model.fc.in_features, 10)
```

### 미세 조정 (Fine-tuning)

사전 학습 모델의 일부 또는 전체 레이어의 가중치를 고정 해제하고, 새로운 데이터셋으로 다시 학습하는 방법입니다. 사전 학습 가중치를 초기값으로 사용해 새로운 도메인에 맞게 미세하게 조정합니다.

일반적으로 초기 레이어는 고정하고 후반 레이어만 학습하거나, 전체 레이어를 매우 작은 학습률로 학습합니다. 초기 레이어의 범용적인 특성은 유지하면서, 후반 레이어의 도메인 특화 특성만 조정하는 전략입니다.

```python
# 전체 파라미터 고정 후 마지막 레이어 교체
for param in model.parameters():
    param.requires_grad = False
model.fc = nn.Linear(model.fc.in_features, 10)

# 후반부(layer4)만 고정 해제
for param in model.layer4.parameters():
    param.requires_grad = True

# 미세 조정 시 매우 작은 학습률 사용
optimizer = torch.optim.Adam([
    {'params': model.layer4.parameters(), 'lr': 1e-4},
    {'params': model.fc.parameters(), 'lr': 1e-3}
])
```

미세 조정은 특성 추출보다 더 많은 데이터가 필요하지만, 사전 학습 도메인과 새 도메인이 달라도 더 높은 성능을 낼 수 있습니다.

## 5. 데이터 크기와 도메인 유사성에 따른 전략 선택

전이 학습 전략은 두 가지 축을 기준으로 선택합니다.

```
                    사전 학습 도메인과 유사
                           ↑
            데이터 적음     │     데이터 적음
            도메인 유사     │     도메인 다름
            → 특성 추출     │     → 후반 레이어만 미세 조정
                           │
적음 ───────────────────────┼─────────────────────── 많음
(데이터)                   │                    (데이터)
            데이터 많음     │     데이터 많음
            도메인 유사     │     도메인 다름
            → 전체 미세 조정│     → 전체 미세 조정
                           ↓
                    사전 학습 도메인과 다름
```

데이터가 적고 도메인이 유사하면 특성 추출로도 충분합니다. 데이터가 많고 도메인이 다르면 전체 미세 조정이 필요합니다. 데이터가 적고 도메인이 많이 다르면 후반 레이어만 선택적으로 미세 조정하는 것이 과적합을 방지하면서 도메인 적응을 가능하게 합니다.

## 6. 전이 학습 적용 시 주의할 점

전이 학습을 적용할 때 빠뜨리기 쉬운 중요한 사항들이 있습니다.

입력 전처리를 사전 학습 데이터셋 기준에 맞춰야 합니다. ImageNet으로 학습된 모델은 ImageNet의 평균과 표준편차로 정규화된 입력을 기대합니다. 이 정규화 과정을 빠뜨리면 사전 학습된 가중치가 의미를 잃습니다.

미세 조정 시 학습률을 처음부터 학습할 때보다 훨씬 작게 설정해야 합니다. 사전 학습된 가중치는 이미 좋은 위치에 있습니다. 큰 학습률로 업데이트하면 오히려 학습된 좋은 특성이 파괴됩니다. 일반적으로 1e-4에서 1e-5 수준의 학습률을 사용합니다.

배치 정규화 레이어의 처리도 중요합니다. 미세 조정 시 배치 정규화 레이어를 평가 모드(eval mode)로 고정하는 경우가 있습니다. 새로운 데이터셋의 배치 크기가 작거나 분포가 다를 때 배치 정규화 통계가 불안정해질 수 있기 때문입니다.

## 핵심 요약

| | 특성 추출 | 미세 조정 |
|---|---|---|
| **방법** | 사전 학습 가중치 고정, 분류 레이어만 학습 | 일부 또는 전체 가중치 재학습 |
| **데이터 요구량** | 적은 데이터로도 가능 | 상대적으로 많은 데이터 필요 |
| **학습 속도** | 빠름 | 느림 |
| **과적합 위험** | 낮음 | 상대적으로 높음 |
| **적합한 상황** | 데이터 적고 도메인 유사 | 데이터 많거나 도메인 다름 |

전이 학습은 데이터가 부족한 현실적인 문제를 해결하는 가장 실용적인 방법입니다. 처음부터 학습하는 것보다 훨씬 적은 데이터와 시간으로 높은 성능을 달성할 수 있으며, 현재 이미지 분류 실무에서 전이 학습 없이 모델을 처음부터 학습시키는 경우는 매우 드뭅니다.
