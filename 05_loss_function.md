# 2. 손실 함수(Loss Function) — 모델이 무엇을 최적화할지를 정의하는 함수

## 1. 핵심 개념: "얼마나 틀렸는가"를 수치화하는 함수

모델이 학습한다는 것은 스스로의 예측이 얼마나 틀렸는지를 인식하고, 그 오차를 줄이는 방향으로 자신을 조정한다는 뜻입니다. 그런데 모델이 "틀렸다"는 사실을 인식하려면, 틀린 정도를 측정할 수 있는 기준이 필요합니다. 손실 함수(Loss Function)는 바로 이 역할을 합니다. 모델의 예측값(ŷ)과 실제 정답(y) 사이의 차이를 하나의 숫자로 환산해주는 함수입니다.

이 숫자가 클수록 예측이 정답에서 많이 벗어났다는 뜻이고, 0에 가까울수록 예측이 정답에 근접했다는 뜻입니다. 모델은 이 숫자를 최소화하는 방향으로 학습합니다. 따라서 손실 함수는 단순한 측정 도구가 아닙니다. **모델이 무엇을 잘하려고 노력하는지를 정의하는 함수**입니다. 손실 함수를 바꾸면 같은 데이터, 같은 모델 구조를 사용해도 모델이 최적화하는 목표 자체가 달라집니다.

직관적으로 이해하자면 양궁 과녁에 비유할 수 있습니다. 실제값(y)은 과녁의 중심이고, 예측값(ŷ)은 화살이 꽂힌 위치입니다. 손실(loss)은 화살과 중심 사이의 거리입니다. 모델은 매 학습마다 "어떻게 쏘면 이 거리가 줄어드는가"를 계산하며 조금씩 조준을 교정합니다.

---

## 2. 학습 과정에서의 역할

손실 함수는 머신러닝 학습의 피드백 루프 한가운데에 위치합니다. 학습이 진행되는 순서를 따라가면 이 역할이 더 명확해집니다.

먼저 모델은 입력 X를 받아 예측값 ŷ을 출력합니다. 그다음 손실 함수가 ŷ과 실제 정답 y를 비교해 하나의 손실값 L을 계산합니다. 이 L이 핵심입니다. 역전파(Backpropagation) 알고리즘은 이 손실값을 모델의 각 파라미터에 대해 미분하여, 각 파라미터가 손실에 얼마나 기여하고 있는지를 계산합니다. 경사 하강법(Gradient Descent)은 이 미분값(기울기)을 이용해 파라미터를 손실이 줄어드는 방향으로 조금씩 이동시킵니다.

```
θ = θ - α × ∂L/∂θ   (α = 학습률, learning rate)
```

손실 함수가 없으면 미분 자체가 불가능하고, 미분이 없으면 역전파도, 경사 하강법도 동작하지 않습니다. 즉, **손실 함수 없이는 학습 자체가 일어나지 않습니다.** 이것이 손실 함수가 머신러닝의 근간에 위치하는 이유입니다.

---

## 3. 회귀(Regression)용 손실 함수

### MSE — Mean Squared Error (평균 제곱 오차)

MSE는 회귀 문제에서 가장 널리 사용되는 손실 함수입니다. 예측값과 실제값의 차이를 제곱한 뒤 평균을 냅니다.

$$L = \frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2$$

제곱을 취하기 때문에 오차가 클수록 패널티가 기하급수적으로 커집니다. 예측이 1만큼 틀리면 손실이 1이지만, 3만큼 틀리면 손실이 9가 됩니다. 이 특성 덕분에 큰 오차를 강하게 억제하지만, 반대로 이상치(outlier)가 존재할 경우 이상치 하나가 전체 손실을 크게 끌어올려 모델이 이상치에 지나치게 집중하게 만드는 부작용도 있습니다. 함수 자체가 수학적으로 매끄럽게(differentiable) 미분 가능하므로 경사 하강법에 잘 맞습니다.

### MAE — Mean Absolute Error (평균 절대 오차)

MAE는 예측값과 실제값의 차이의 절댓값을 평균냅니다.

$$L = \frac{1}{n} \sum_{i=1}^{n} |y_i - \hat{y}_i|$$

오차에 제곱을 취하지 않으므로 모든 오차를 동등하게 취급합니다. 이상치가 있어도 그 영향이 MSE처럼 증폭되지 않아 강건(robust)하다는 장점이 있습니다. 그러나 오차가 0인 지점에서 미분이 불연속이라는 수학적 한계가 있어, 학습이 불안정해질 수 있습니다.

### Huber Loss

Huber Loss는 MSE와 MAE의 장점을 결합한 절충안입니다. 오차가 특정 임계값(δ)보다 작을 때는 MSE처럼 동작해 미분 안정성을 유지하고, 오차가 임계값보다 클 때는 MAE처럼 동작해 이상치에 강건하게 반응합니다. 이상치가 존재하면서도 학습 안정성이 필요한 실무 상황에서 자주 선택됩니다.

| | MSE | MAE | Huber |
|---|---|---|---|
| 이상치 민감도 | 높음 | 낮음 | 중간 |
| 미분 연속성 | 연속 | 0에서 불연속 | 연속 |
| 사용 상황 | 이상치 적을 때 | 이상치 많을 때 | 혼합 상황 |

## 4. 분류(Classification)용 손실 함수

### Binary Cross-Entropy (이진 교차 엔트로피)

이진 분류, 즉 출력이 두 가지 클래스(0 또는 1)인 문제에 사용합니다. 모델이 출력하는 것은 0~1 사이의 확률값입니다. 이 손실 함수는 모델이 정답 클래스에 얼마나 높은 확률을 부여했는가를 기준으로 손실을 계산합니다.

$$L = -\frac{1}{n}\sum_{i=1}^{n} [y_i \log(\hat{y}_i) + (1-y_i)\log(1-\hat{y}_i)]$$

직관적으로 이해하면, 모델이 "90% 확률로 양성"이라고 예측했는데 실제로 양성이면 손실이 작습니다. 반대로 "90% 확률로 양성"이라고 했는데 실제로 음성이라면 손실이 매우 크게 계산됩니다. 즉, 확신이 강한 틀린 예측일수록 더 큰 페널티를 부여하는 구조입니다. 이 설계가 모델로 하여금 섣불리 높은 확률을 부여하지 않도록 학습하게 만듭니다.

### Categorical Cross-Entropy (다중 클래스 교차 엔트로피)

출력 클래스가 3개 이상인 문제에 사용합니다. 모델 마지막 층에 Softmax 활성화 함수를 붙여 각 클래스에 대한 확률 분포를 출력하고, 이 분포와 실제 정답 클래스 사이의 교차 엔트로피를 손실로 계산합니다. 정답 클래스에 부여된 확률이 높을수록 손실이 줄어드는 방식으로 작동합니다.

## 5. 손실 함수 선택이 왜 중요한가

손실 함수 선택은 단순한 기술적 선택이 아니라 **"모델에게 무엇이 실수인가"를 정의하는 설계 결정**입니다. 같은 데이터와 모델 구조를 사용하더라도, 손실 함수가 달라지면 모델이 완전히 다른 목표를 향해 학습합니다.

예를 들어 암 진단 모델에서 단순히 정확도(Accuracy)를 최적화하면, 양성 환자가 전체의 5%뿐인 불균형 데이터에서 모델은 모든 환자를 "음성"으로 예측해도 95%의 정확도를 달성할 수 있습니다. 수치상으로는 훌륭해 보이지만, 실제로는 환자를 한 명도 찾아내지 못하는 쓸모없는 모델입니다. 이 경우 재현율(Recall)을 직접 최적화하거나, 클래스 불균형을 고려한 손실 함수를 설계해야 합니다.

이상치가 많은 부동산 가격 예측에서 MSE를 사용하면, 몇 개의 극단적으로 비싼 매물이 손실을 지배하게 되어 일반적인 매물 예측은 오히려 부정확해집니다. 이런 경우 MAE나 Huber Loss가 더 적합합니다. 손실 함수는 비즈니스 목표와 데이터 특성을 함께 고려해서 선택해야 합니다.

## 6. 손실 함수와 경사 하강법의 관계

손실 함수를 파라미터 공간에 시각화하면 구불구불한 지형처럼 보입니다. 어느 지점은 높고 어느 지점은 낮으며, 가장 낮은 곳이 손실이 최소화되는 지점, 즉 모델이 도달하고자 하는 목표입니다.

```
손실(L)
  │        ╲        /
  │         ╲      /
  │          ╲    /
  │           ╲__/   ← 전역 최솟값(Global Minimum)
  └──────────────────── 파라미터(θ)
```

경사 하강법은 이 지형에서 가장 가파른 내리막 방향으로 한 걸음씩 이동하는 알고리즘입니다. 학습률(Learning Rate, α)은 한 번에 이동하는 보폭입니다. 보폭이 너무 크면 최솟값을 지나쳐 반대편으로 넘어가며 진동하고, 보폭이 너무 작으면 수렴하는 데 너무 오래 걸리거나 지역 최솟값(Local Minimum)에 갇히게 됩니다.

MSE처럼 볼록(convex)한 손실 함수는 지형에 계곡이 하나뿐이어서 어디서 출발해도 전역 최솟값(Global Minimum)에 도달이 보장됩니다. 반면 신경망에서 사용되는 복잡한 손실 함수는 지형이 복잡해 여러 개의 지역 최솟값이 존재하고, 항상 전역 최솟값에 도달한다는 보장이 없습니다.

---

## 7. 실무 선택 가이드

```
문제 유형
├── 회귀
│   ├── 이상치 적음         → MSE
│   ├── 이상치 많음         → MAE 또는 Huber Loss
│   └── 불확실성 추정 필요  → NLL (Negative Log-Likelihood)
│
└── 분류
    ├── 이진 분류           → Binary Cross-Entropy
    ├── 다중 클래스         → Categorical Cross-Entropy
    └── 다중 레이블         → Binary Cross-Entropy (클래스별 독립 적용)
```

| 개념 | 내용 |
|---|---|
| **정의** | 예측값과 실제값의 차이를 수치화한 함수 |
| **역할** | 모델이 학습할 방향을 결정하는 나침반 |
| **중요성** | 손실 함수 없이는 역전파·경사 하강법 불가 → 학습 자체가 불가 |
| **선택 기준** | 문제 유형(회귀/분류) + 데이터 특성(이상치 여부) + 비즈니스 목표 |

손실 함수는 모델에게 "무엇이 실수인가"를 가르치는 함수입니다. 잘못된 손실 함수를 선택하면, 모델은 틀린 것을 완벽하게 최적화합니다. 반대로 문제에 맞는 손실 함수를 정확히 설계하면, 모델은 비즈니스 목표와 정렬된 방향으로 스스로를 개선합니다.
