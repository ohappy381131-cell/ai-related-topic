## 1. 부스팅이란: 약한 학습기를 순차적으로 강화하는 방법

부스팅(Boosting)은 여러 개의 약한 학습기(weak learner)를 순차적으로 학습시켜 하나의 강한 학습기(strong learner)를 만드는 앙상블 기법입니다. 여기서 약한 학습기란 무작위 추측보다 조금 더 나은 수준의 모델을 뜻하며, 대부분의 경우 깊이가 얕은 결정 트리(depth 1~3)를 사용합니다.

부스팅의 핵심 아이디어는 **이전 모델이 틀린 것에 다음 모델이 집중한다**는 것입니다. 첫 번째 모델을 학습시키고 나면, 그 모델이 잘못 예측한 데이터에 더 높은 가중치를 부여합니다. 두 번째 모델은 이 가중치가 반영된 데이터로 학습하므로, 첫 번째 모델이 틀린 부분을 더 잘 맞추는 방향으로 훈련됩니다. 이 과정을 반복하면서 각 모델이 이전 모델의 약점을 보완하는 구조가 만들어집니다.

이 점이 Bagging과 가장 결정적으로 다른 부분입니다. Bagging은 여러 모델을 병렬로 독립적으로 학습시키고 결과를 평균 내어 분산을 줄입니다. 반면 부스팅은 모델들이 순차적으로 의존하며 학습하고, 이전 모델의 오차를 다음 모델이 보정하는 방식으로 편향을 줄입니다.

```
Bagging:   모델1 ─┐
           모델2 ─┼─ 평균 → 최종 예측  (병렬, 분산 감소)
           모델3 ─┘

Boosting:  모델1 → 오차 → 모델2 → 오차 → 모델3 → 합산 → 최종 예측  (순차, 편향 감소)
```

## 2. AdaBoost (Adaptive Boosting)

AdaBoost는 1997년 Freund와 Schapire가 제안한 최초의 실용적인 부스팅 알고리즘입니다. 부스팅이라는 개념을 처음으로 구체화한 모델로, 이후 등장하는 모든 부스팅 계열 알고리즘의 이론적 출발점입니다.

AdaBoost의 작동 방식은 다음과 같습니다. 처음에는 모든 훈련 샘플에 동일한 가중치를 부여합니다. 첫 번째 약한 학습기를 학습시킨 뒤, 잘못 분류된 샘플의 가중치를 높이고 올바르게 분류된 샘플의 가중치를 낮춥니다. 두 번째 학습기는 이 가중치가 반영된 데이터로 학습하므로 이전에 틀린 샘플에 더 집중하게 됩니다. 이 과정을 지정한 횟수만큼 반복한 뒤, 각 학습기의 예측을 정확도에 비례한 가중치로 합산해 최종 예측을 만듭니다.

**장점**으로는 구현이 단순하고 직관적이며, 과적합에 비교적 강한 편입니다. 또한 이진 분류 문제에서 이론적으로 훈련 오차가 0에 수렴함이 증명되어 있습니다. 특성 스케일링이 필요 없고, 약한 학습기만으로도 충분한 성능을 낼 수 있습니다.

**단점**으로는 이상치(outlier)에 민감합니다. 잘못 분류된 샘플에 가중치를 높이는 방식이기 때문에, 이상치가 반복적으로 높은 가중치를 받아 모델 전체를 왜곡할 수 있습니다. 또한 노이즈가 많은 데이터에서 성능이 급격히 떨어지고, Gradient Boosting 계열에 비해 예측 성능이 낮은 경우가 많습니다.

## 3. Gradient Boosting

Gradient Boosting은 2001년 Jerome Friedman이 제안한 알고리즘으로, 부스팅을 경사 하강법(Gradient Descent)의 관점에서 재해석한 모델입니다. AdaBoost가 샘플 가중치를 조정하는 방식으로 오차를 보정했다면, Gradient Boosting은 이전 모델의 잔차(residual), 즉 실제값과 예측값의 차이를 직접 다음 모델이 학습하는 방식으로 작동합니다.

수학적으로는 손실 함수의 음의 그래디언트(negative gradient)를 계산하고, 다음 모델이 이 그래디언트를 예측하도록 학습합니다. 이 덕분에 손실 함수를 MSE, Cross-Entropy 등 다양하게 교체할 수 있어 회귀와 분류 모두에 유연하게 적용할 수 있습니다.

**장점**으로는 다양한 손실 함수를 지원해 문제에 맞게 최적화 목표를 설정할 수 있습니다. AdaBoost보다 이상치에 더 강건하고, 예측 성능도 일반적으로 더 높습니다. 하이퍼파라미터 조정을 통해 과적합을 어느 정도 제어할 수 있습니다.

**단점**으로는 순차 학습 특성상 병렬화가 어렵고 학습 속도가 느립니다. 트리 수, 학습률, 최대 깊이 등 하이퍼파라미터가 많아 튜닝에 시간이 걸립니다. 대규모 데이터셋에서는 계산 비용이 상당합니다.

---

## 4. XGBoost (Extreme Gradient Boosting)

XGBoost는 2016년 Chen과 Guestrin이 발표한 알고리즘으로, Gradient Boosting의 성능과 속도를 대폭 개선한 모델입니다. Kaggle을 비롯한 데이터 사이언스 경진대회에서 압도적인 성과를 보이며 빠르게 표준 알고리즘으로 자리 잡았습니다.

Gradient Boosting과 구조적으로 유사하지만 몇 가지 핵심적인 개선이 있습니다. 손실 함수에 L1, L2 규제 항을 명시적으로 추가해 과적합을 억제합니다. 트리를 노드 단위로 병렬 계산하고, 데이터를 메모리에 효율적으로 캐싱해 학습 속도를 크게 높였습니다. 결측값을 자동으로 처리하는 기능도 내장되어 있습니다. 또한 조기 종료(early stopping) 기능을 지원해, 검증 성능이 개선되지 않으면 학습을 자동으로 중단합니다.

**장점**으로는 Gradient Boosting 대비 학습 속도가 빠르고, 내장된 규제 덕분에 과적합 제어가 용이합니다. 결측값 처리, 조기 종료, 특성 중요도 제공 등 실무에서 유용한 기능이 풍부하게 내장되어 있습니다. 정형 데이터에서 최고 수준의 예측 성능을 보입니다.

**단점**으로는 하이퍼파라미터 수가 많아 최적 설정을 찾는 데 시간이 필요합니다. 대용량 데이터에서 메모리 사용량이 높을 수 있고, 딥러닝 모델에 비해 비정형 데이터(이미지, 텍스트)에는 적합하지 않습니다.

---

## 5. LightGBM (Light Gradient Boosting Machine)

LightGBM은 2017년 Microsoft가 개발한 알고리즘으로, XGBoost보다 학습 속도를 더욱 높이고 메모리 사용량을 줄이는 데 초점을 맞춘 모델입니다. 대용량 데이터셋에서 XGBoost가 느리다는 한계를 극복하기 위해 두 가지 핵심 기술을 도입했습니다.

첫 번째는 GOSS(Gradient-based One-Side Sampling)입니다. 기울기(gradient)가 큰, 즉 오차가 큰 샘플은 전부 유지하고, 기울기가 작은 샘플은 일부만 무작위로 추출해 사용합니다. 정보 손실을 최소화하면서 데이터 양을 줄여 학습 속도를 높입니다.

두 번째는 EFB(Exclusive Feature Bundling)입니다. 동시에 0이 아닌 값을 갖지 않는 희소(sparse) 특성들을 하나로 묶어 특성 수를 효과적으로 줄입니다. 고차원 희소 데이터에서 특히 효과적입니다.

또한 XGBoost가 트리를 레벨(level) 단위로 수평 확장하는 것과 달리, LightGBM은 리프(leaf) 단위로 수직 확장합니다. 손실이 가장 큰 리프를 우선적으로 분기하므로 같은 리프 수에서 더 낮은 손실을 달성할 수 있습니다.

**장점**으로는 XGBoost 대비 학습 속도가 수 배 빠르고 메모리 사용량이 적습니다. 대용량 데이터셋과 고차원 데이터에 강하며, 범주형 변수를 직접 처리하는 기능도 지원합니다. 예측 성능도 XGBoost와 대등하거나 경우에 따라 더 높습니다.

**단점**으로는 리프 단위 확장 방식으로 인해 데이터가 적을 때 과적합 위험이 XGBoost보다 높습니다. 하이퍼파라미터 튜닝이 민감해 잘못 설정하면 성능이 크게 낮아질 수 있습니다.

---

## 6. CatBoost (Categorical Boosting)

CatBoost는 2017년 러시아 검색엔진 기업 Yandex가 개발한 알고리즘입니다. 이름에서 드러나듯 범주형(categorical) 변수 처리에 특화되어 있습니다.

기존 부스팅 모델들은 범주형 변수를 처리하기 위해 원-핫 인코딩(One-Hot Encoding)이나 레이블 인코딩(Label Encoding) 같은 전처리가 필요합니다. CatBoost는 이 과정을 내부적으로 자동 처리하며, 타깃 통계(target statistics) 기반의 정교한 인코딩 방식을 사용해 범주형 변수의 정보를 더 효과적으로 활용합니다.

또한 순서화된 부스팅(Ordered Boosting)이라는 독자적인 학습 방식을 도입했습니다. 기존 Gradient Boosting 계열에서 잔차를 계산할 때 발생하는 예측 편향(prediction shift) 문제를 해결하기 위해, 각 샘플의 잔차를 해당 샘플 이전 데이터만으로 계산합니다. 이 방식이 과적합을 억제하고 일반화 성능을 높이는 효과가 있습니다.

**장점**으로는 범주형 변수가 많은 데이터에서 전처리 없이 바로 사용할 수 있어 편의성이 높습니다. 기본 하이퍼파라미터로도 좋은 성능을 내는 경우가 많아 튜닝 부담이 적습니다. 과적합 제어 성능이 우수하고, GPU 학습도 지원합니다.

**단점**으로는 학습 속도가 XGBoost, LightGBM에 비해 느린 경우가 있습니다. 범주형 변수가 적은 순수 수치형 데이터에서는 LightGBM 대비 이점이 줄어듭니다. 메모리 사용량도 상대적으로 높을 수 있습니다.

## 7. 네 가지 부스팅 모델 종합 비교

| | AdaBoost | Gradient Boosting | XGBoost | LightGBM | CatBoost |
|---|---|---|---|---|---|
| **출시** | 1997 | 2001 | 2016 | 2017 | 2017 |
| **오차 보정 방식** | 샘플 가중치 조정 | 잔차 직접 학습 | 잔차 + 규제 | 잔차 + 샘플링 | 잔차 + 순서화 |
| **학습 속도** | 빠름 | 느림 | 중간 | 매우 빠름 | 중간~느림 |
| **이상치 민감도** | 높음 | 중간 | 낮음 | 낮음 | 낮음 |
| **범주형 변수** | 수동 인코딩 | 수동 인코딩 | 수동 인코딩 | 부분 지원 | 자동 처리 |
| **과적합 제어** | 중간 | 중간 | 우수 | 중간 | 우수 |
| **대용량 데이터** | 부적합 | 부적합 | 적합 | 매우 적합 | 적합 |
| **튜닝 난이도** | 낮음 | 중간 | 높음 | 높음 | 낮음 |

## 8. 상황별 선택 가이드

부스팅 모델 선택은 데이터의 특성과 주어진 제약 조건에 따라 달라집니다.

범주형 변수가 많고 전처리를 최소화하고 싶다면 CatBoost가 가장 자연스러운 선택입니다. 대용량 데이터에서 학습 속도가 중요하다면 LightGBM이 유리합니다. 성능과 안정성의 균형이 중요하고 다양한 레퍼런스가 필요하다면 XGBoost가 검증된 선택입니다. 간단한 문제에서 빠르게 기준 모델(baseline)을 잡고 싶다면 AdaBoost도 여전히 유효합니다.

실무에서는 세 가지 모델(XGBoost, LightGBM, CatBoost)을 모두 학습시켜 보고 교차 검증 성능을 비교한 뒤 선택하는 것이 일반적입니다. 더 나아가 세 모델의 예측을 앙상블하면 단일 모델보다 더 안정적인 성능을 얻는 경우도 많습니다.

---

## 핵심 요약

부스팅은 약한 학습기를 순차적으로 쌓아 이전 모델의 오차를 다음 모델이 보정하는 방식으로 편향을 줄이는 앙상블 기법입니다. AdaBoost가 이 개념을 처음 실용화했고, Gradient Boosting이 경사 하강법 관점으로 확장했으며, XGBoost·LightGBM·CatBoost는 각각 속도, 대용량 처리, 범주형 변수 처리라는 실무적 한계를 극복하며 발전해 왔습니다. 현재 정형 데이터 분야에서 이 세 모델은 딥러닝을 포함한 어떤 모델과 비교해도 경쟁력 있는 최상위 알고리즘입니다.
