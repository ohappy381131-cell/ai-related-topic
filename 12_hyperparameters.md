# 9. 딥러닝 하이퍼파라미터 — 성능 향상을 위해 고려해야 할 종류와 역할

본 글에서는 딥러닝 모델의 성능에 영향을 미치는 하이퍼파라미터의 종류와 각각의 역할을 하나의 흐름 속에서 정리합니다. 파라미터와의 차이부터 시작해 학습 과정, 모델 구조, 과적합 방지, 옵티마이저 관련 하이퍼파라미터까지, 실무에서 어떤 기준으로 조정하는지를 구조적으로 이해할 수 있도록 설명합니다.

## 1. 하이퍼파라미터란: 파라미터와의 차이

딥러닝 모델에는 두 종류의 값이 존재합니다. 하나는 학습 과정에서 데이터를 통해 자동으로 업데이트되는 **파라미터(parameter)**이고, 다른 하나는 학습이 시작되기 전에 사람이 직접 설정해야 하는 **하이퍼파라미터(hyperparameter)**입니다.

가중치(weight)와 편향(bias)이 파라미터의 대표적인 예입니다. 이 값들은 역전파와 경사 하강법을 통해 모델이 스스로 조정합니다. 반면 학습률, 배치 크기, 층의 수, 에포크 수 같은 값들은 모델이 스스로 결정하지 못합니다. 이것들이 하이퍼파라미터이며, 이 값들을 어떻게 설정하느냐에 따라 같은 모델 구조와 데이터를 사용해도 성능이 크게 달라집니다.

하이퍼파라미터 튜닝(hyperparameter tuning)은 최적의 하이퍼파라미터 조합을 찾는 과정으로, 딥러닝 실무에서 상당한 시간과 자원이 투입되는 작업입니다.

---

## 2. 학습 과정 관련 하이퍼파라미터

### 학습률 (Learning Rate, lr)

학습률은 딥러닝에서 가장 중요한 하이퍼파라미터 중 하나입니다. 경사 하강법에서 파라미터를 업데이트할 때 한 번에 얼마나 크게 이동할지를 결정합니다.

```
θ = θ - lr × ∂L/∂θ
```

학습률이 너무 크면 손실 함수의 최솟값을 지나쳐 발산하거나 진동합니다. 너무 작으면 수렴 속도가 극도로 느려지고 지역 최솟값(local minimum)에 갇힐 수 있습니다. 일반적으로 0.1, 0.01, 0.001 같은 값에서 시작해 조정하며, 0.001이 많은 경우에서 좋은 출발점이 됩니다.

실무에서는 고정된 학습률 대신 학습 과정에서 학습률을 동적으로 조절하는 **학습률 스케줄러(learning rate scheduler)**를 함께 사용합니다. 초반에는 큰 학습률로 빠르게 수렴하고, 후반에는 작은 학습률로 세밀하게 조정하는 방식입니다. Step Decay, Cosine Annealing, Warm-up 등 다양한 스케줄링 전략이 있습니다.

### 배치 크기 (Batch Size)

전체 훈련 데이터를 한 번에 모두 사용해 기울기를 계산하는 것은 메모리와 계산 측면에서 비현실적입니다. 배치 크기는 한 번의 파라미터 업데이트에 사용하는 샘플의 수입니다.

배치 크기가 크면 기울기 추정이 안정적이고 병렬 계산에 유리하지만, 메모리 사용량이 많고 지역 최솟값에 빠질 가능성이 높아집니다. 배치 크기가 작으면 기울기에 노이즈가 많아지지만, 이 노이즈가 오히려 지역 최솟값을 탈출하는 데 도움이 되기도 합니다. 또한 메모리 효율이 좋습니다. 일반적으로 32, 64, 128, 256처럼 2의 거듭제곱 값을 사용하며, GPU 메모리 크기에 맞게 조정합니다.

### 에포크 수 (Epochs)

에포크는 전체 훈련 데이터를 한 번 모두 사용해 학습하는 단위입니다. 에포크 수는 훈련 데이터 전체를 몇 번 반복해서 학습할지를 결정합니다.

에포크가 너무 적으면 모델이 충분히 수렴하지 못하고, 너무 많으면 과적합이 발생합니다. 이 때문에 고정된 에포크 수 대신 검증 손실이 개선되지 않을 때 학습을 자동으로 멈추는 **조기 종료(Early Stopping)**를 함께 사용하는 것이 일반적입니다.

## 3. 모델 구조 관련 하이퍼파라미터

### 은닉층의 수 (Number of Hidden Layers)

은닉층이 많을수록 모델이 더 복잡한 패턴을 표현할 수 있지만, 학습이 어려워지고 과적합 위험도 커집니다. 층이 너무 적으면 표현력이 부족해 과소적합이 발생합니다. 문제의 복잡도에 맞게 설정해야 하며, 일반적으로 단순한 문제에서는 2~3층, 복잡한 문제에서는 더 깊은 구조를 사용합니다.

### 은닉층의 뉴런 수 (Number of Neurons per Layer)

각 층에 몇 개의 뉴런을 배치할지를 결정합니다. 뉴런이 많을수록 표현력이 높아지지만 계산 비용과 과적합 위험이 증가합니다. 일반적으로 입력층에서 출력층 방향으로 점차 줄어드는 피라미드 구조(예: 512 → 256 → 128)가 자주 사용됩니다.

### 활성화 함수 (Activation Function)

각 뉴런에서 사용할 비선형 함수를 선택합니다. 은닉층에서는 ReLU가 가장 널리 사용되며, 기울기 소실(vanishing gradient) 문제를 완화합니다. ReLU의 변형인 Leaky ReLU, ELU, GELU도 상황에 따라 사용됩니다. 출력층에서는 문제 유형에 따라 이진 분류에는 Sigmoid, 다중 분류에는 Softmax, 회귀에는 선형 활성화 함수를 사용합니다.

## 4. 과적합 방지 관련 하이퍼파라미터

### 드롭아웃 비율 (Dropout Rate)

드롭아웃(Dropout)은 학습 중 무작위로 일부 뉴런을 비활성화해 특정 뉴런에 대한 의존도를 낮추는 정규화 기법입니다. 드롭아웃 비율은 비활성화할 뉴런의 비율을 결정합니다. 0.2~0.5가 일반적이며, 값이 클수록 규제가 강해집니다. 너무 높으면 모델의 표현력 자체가 낮아져 과소적합이 발생할 수 있습니다.

### 규제 강도 (Regularization Strength, λ)

L1 또는 L2 규제를 적용할 때 그 강도를 결정하는 값입니다. λ가 클수록 파라미터 크기에 강한 페널티를 부여해 모델이 단순해지고, 작을수록 규제 효과가 약해집니다. 과적합이 심할 때 λ를 높이고, 과소적합이 발생하면 낮춥니다.

### 배치 정규화 (Batch Normalization)

배치 정규화는 각 층의 출력을 미니배치 단위로 정규화하는 기법으로, 학습 안정성을 높이고 더 큰 학습률 사용을 가능하게 합니다. 배치 정규화를 적용할 층의 위치가 하이퍼파라미터가 됩니다.

---

## 5. 옵티마이저 관련 하이퍼파라미터

옵티마이저(optimizer)는 기울기를 사용해 파라미터를 업데이트하는 알고리즘입니다. 어떤 옵티마이저를 선택하느냐, 그리고 그 옵티마이저의 세부 설정값이 하이퍼파라미터가 됩니다.

**SGD(Stochastic Gradient Descent)**는 가장 기본적인 옵티마이저로, 학습률과 모멘텀(momentum)이 주요 하이퍼파라미터입니다. 모멘텀은 이전 기울기 방향을 현재 업데이트에 반영해 진동을 줄이고 수렴을 가속합니다.

**Adam(Adaptive Moment Estimation)**은 현재 가장 널리 사용되는 옵티마이저입니다. 각 파라미터마다 적응적으로 학습률을 조정합니다. 학습률 외에도 β₁(1차 모멘트 감쇠율, 기본값 0.9), β₂(2차 모멘트 감쇠율, 기본값 0.999), ε(수치 안정성을 위한 작은 값)가 하이퍼파라미터입니다. 대부분의 경우 기본값으로도 잘 작동하므로, 실무에서는 학습률만 조정하는 경우가 많습니다.

| 옵티마이저 | 특징 | 주요 하이퍼파라미터 |
|---|---|---|
| SGD | 단순, 일반화 성능 우수 | lr, momentum |
| Adam | 빠른 수렴, 범용적 | lr, β₁, β₂, ε |
| RMSprop | 시계열에 강함 | lr, decay |
| AdaGrad | 희소 데이터에 강함 | lr |

## 6. 하이퍼파라미터 탐색 전략

하이퍼파라미터의 최적 조합을 찾는 방법도 여러 가지입니다.

**그리드 탐색(Grid Search)**은 지정한 모든 하이퍼파라미터 조합을 순차적으로 시도합니다. 완전 탐색이므로 최적값을 놓칠 가능성이 없지만, 하이퍼파라미터 수가 늘어날수록 계산 비용이 기하급수적으로 증가합니다.

**랜덤 탐색(Random Search)**은 하이퍼파라미터 공간에서 무작위로 조합을 샘플링합니다. 그리드 탐색보다 효율적으로 넓은 공간을 탐색할 수 있으며, 연구에 따르면 많은 경우 그리드 탐색과 비슷하거나 더 좋은 결과를 냅니다.

**베이지안 최적화(Bayesian Optimization)**는 이전 시도의 결과를 바탕으로 다음에 시도할 조합을 확률적으로 선택합니다. 가장 유망한 영역을 집중적으로 탐색하므로 효율이 높습니다. Optuna, Hyperopt 같은 라이브러리가 이 방식을 구현합니다.

## 핵심 요약

| 범주 | 주요 하이퍼파라미터 | 핵심 역할 |
|---|---|---|
| **학습 과정** | 학습률, 배치 크기, 에포크 수 | 학습 속도와 안정성 결정 |
| **모델 구조** | 층 수, 뉴런 수, 활성화 함수 | 모델 표현력 결정 |
| **과적합 방지** | 드롭아웃 비율, 규제 강도(λ) | 일반화 성능 결정 |
| **옵티마이저** | 옵티마이저 종류, β₁, β₂ | 파라미터 업데이트 방식 결정 |

하이퍼파라미터 튜닝에는 정해진 공식이 없습니다. 학습률과 배치 크기처럼 영향이 큰 것부터 먼저 조정하고, 학습 곡선을 관찰하며 과소적합인지 과적합인지를 진단한 뒤, 그에 맞는 하이퍼파라미터를 체계적으로 조정해나가는 것이 실용적인 접근입니다.
