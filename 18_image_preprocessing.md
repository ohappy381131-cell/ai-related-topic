# 18. 이미지 전처리 — 리사이징과 정규화를 하는 이유

본 글에서는 이미지를 딥러닝 모델에 입력하기 전에 리사이징(Resizing)과 정규화(Normalization)를 수행하는 이유를 하나의 흐름 속에서 정리합니다. 두 전처리 과정이 각각 어떤 문제를 해결하고, 어떤 방식으로 작동하며, 올바르게 적용하지 않으면 어떤 문제가 생기는지를 구조적으로 이해할 수 있도록 설명합니다.

---

## 목차

1. 왜 이미지를 그대로 모델에 넣으면 안 되는가
2. 리사이징 (Resizing)

---

## 1. 왜 이미지를 그대로 모델에 넣으면 안 되는가

카메라로 찍은 사진, 스마트폰으로 찍은 이미지, 웹에서 수집한 이미지는 모두 크기와 픽셀값의 범위가 제각각입니다. 어떤 이미지는 4032×3024 픽셀이고, 어떤 이미지는 640×480 픽셀입니다. 픽셀값은 0부터 255까지의 정수로 표현됩니다.

이 상태 그대로 딥러닝 모델에 넣으면 두 가지 근본적인 문제가 생깁니다. 첫째, 모델은 고정된 크기의 입력을 기대합니다. 크기가 다른 이미지들을 같은 모델에 넣을 수 없습니다. 둘째, 픽셀값의 범위와 분포가 학습을 불안정하게 만듭니다. 리사이징과 정규화는 이 두 문제를 각각 해결하기 위한 전처리 과정입니다.

---

## 2. 리사이징 (Resizing)

### 왜 필요한가

딥러닝 모델은 구조가 고정되어 있습니다. 입력층의 뉴런 수, 합성곱 레이어의 필터 크기, 완전 연결 레이어의 파라미터 수가 모두 특정 입력 크기를 기준으로 설계됩니다. 예를 들어 ResNet-50은 224×224 크기의 이미지를 입력으로 받도록 설계되어 있습니다. 다른 크기의 이미지를 넣으면 레이어 간 차원이 맞지 않아 연산 자체가 불가능합니다.

또한 배치 학습(batch training)을 위해서도 크기가 통일되어야 합니다. GPU는 동일한 크기의 이미지 여러 장을 묶어 병렬로 처리할 때 효율이 극대화됩니다. 크기가 다른 이미지들을 하나의 배치로 묶으려면 패딩(padding)이나 리사이징이 필요합니다.

### 어떻게 작동하는가

리사이징은 원본 이미지의 픽셀을 보간(interpolation) 방법으로 재배치해 목표 크기의 이미지를 만드는 과정입니다. 보간 방법에는 여러 가지가 있습니다. 최근접 이웃 보간(Nearest Neighbor)은 가장 가까운 픽셀값을 그대로 복사해 빠르지만 품질이 낮습니다. 쌍선형 보간(Bilinear Interpolation)은 주변 4개 픽셀의 가중 평균을 사용해 더 부드러운 결과를 냅니다. 쌍삼차 보간(Bicubic Interpolation)은 주변 16개 픽셀을 사용해 더 세밀하지만 계산 비용이 높습니다. 실무에서는 쌍선형 보간이 속도와 품질의 균형으로 가장 많이 쓰입니다.

### 주의할 점

단순히 크기를 줄이는 것은 정보 손실을 수반합니다. 4032×3024 이미지를 224×224로 줄이면 원본 픽셀의 대부분이 버려집니다. 이 과정에서 작은 객체나 세밀한 텍스처가 사라질 수 있습니다. 또한 이미지의 가로세로 비율(aspect ratio)을 무시하고 단순히 늘리거나 줄이면 객체가 찌그러집니다. 이를 방지하기 위해 비율을 유지한 채 크기를 조정한 뒤 나머지 영역을 패딩으로 채우거나, 중앙을 기준으로 잘라내는(center crop) 방식을 사용합니다.

---

## 3. 정규화 (Normalization)

### 왜 필요한가

이미지의 픽셀값은 기본적으로 0부터 255 사이의 정수입니다. 이 범위의 값을 그대로 신경망에 입력하면 세 가지 문제가 발생합니다.

**첫째, 경사 하강법이 불안정해집니다.** 픽셀값이 0~255처럼 크면 가중치 업데이트 시 기울기가 매우 커지거나 작아져 학습이 진동하거나 발산할 수 있습니다. 값의 범위가 작고 균일할수록 경사 하강법이 안정적으로 수렴합니다.

**둘째, 학습 속도가 느려집니다.** 입력값의 범위가 크면 손실 함수의 지형이 비대칭적으로 늘어진 형태가 됩니다. 이런 지형에서 경사 하강법은 지그재그로 느리게 수렴합니다. 입력을 정규화하면 손실 함수의 지형이 더 등방적(isotropic)이 되어 수렴이 빨라집니다.

**셋째, 활성화 함수의 포화(saturation) 문제가 생깁니다.** Sigmoid나 tanh 같은 활성화 함수는 입력값이 너무 크거나 작으면 기울기가 거의 0이 되는 포화 영역에 빠집니다. 입력을 적절한 범위로 정규화하면 활성화 함수가 민감하게 반응하는 구간에서 동작하게 됩니다.

### 두 가지 정규화 방식

**Min-Max 정규화 (0~1 스케일링)**

픽셀값을 0과 1 사이로 선형 변환합니다.

```
정규화된 픽셀 = 원본 픽셀 / 255.0
```

가장 단순한 방법으로, 이미지의 모든 픽셀값이 0~1 사이가 됩니다. 많은 모델에서 기본 전처리로 사용됩니다.

**표준화 (Standardization, Z-score 정규화)**

픽셀값에서 데이터셋 전체의 평균을 빼고 표준편차로 나눕니다.

```
표준화된 픽셀 = (픽셀 / 255.0 - 평균) / 표준편차
```

ImageNet 데이터셋으로 사전 학습된 모델(ResNet, VGG, EfficientNet 등)을 사용할 때는 반드시 ImageNet의 평균과 표준편차로 표준화해야 합니다.

```
ImageNet 평균: [0.485, 0.456, 0.406]  (R, G, B 채널별)
ImageNet 표준편차: [0.229, 0.224, 0.225]
```

이 값들이 특별한 이유는 모델이 이 분포를 기준으로 학습되었기 때문입니다. 다른 분포로 입력하면 사전 학습된 가중치가 의미를 잃어버립니다. 전이 학습(transfer learning)에서 이 표준화 과정을 빠뜨리면 성능이 크게 떨어지는 이유가 여기에 있습니다.

### 채널별 정규화가 필요한 이유

RGB 이미지는 Red, Green, Blue 세 채널로 구성됩니다. 각 채널의 픽셀값 분포가 서로 다를 수 있습니다. 야외 사진은 하늘 때문에 Blue 채널 값이 높고, 실내 사진은 조명 때문에 Red 채널 값이 높을 수 있습니다. 채널별로 독립적으로 정규화하면 각 채널의 분포 차이를 제거해 모델이 색상 편향 없이 패턴을 학습할 수 있습니다.

## 4. 리사이징과 정규화의 순서

실무에서 이미지 전처리는 일반적으로 다음 순서로 진행됩니다.

```
원본 이미지 (가변 크기, 픽셀값 0~255)
    ↓
리사이징 → 고정 크기 (예: 224×224)
    ↓
[선택] 데이터 증강 (훈련 시에만: 랜덤 크롭, 플립, 회전 등)
    ↓
텐서 변환 → 픽셀값 0~255 정수 → 0~1 실수
    ↓
정규화 → 채널별 평균·표준편차 적용
    ↓
모델 입력
```

리사이징을 정규화보다 먼저 하는 이유는 정수 단위 픽셀 연산이 부동소수점 연산보다 효율적이기 때문입니다. 크기를 먼저 줄여 데이터양을 줄인 뒤 정규화하는 것이 계산 효율상 유리합니다.

---

## 5. 정규화를 하지 않으면 어떻게 되는가

정규화 없이 0~255 범위의 픽셀값을 그대로 사용하면 실제로 어떤 일이 일어나는지 구체적으로 짚어보겠습니다. 학습 초기 가중치는 보통 0 근처의 작은 값으로 초기화됩니다. 그런데 입력값이 수백 단위라면 첫 번째 레이어의 출력이 매우 큰 값이 됩니다. 이 큰 값이 활성화 함수를 포화 영역으로 밀어 넣고, 역전파 시 기울기가 거의 0이 되어 학습이 멈추는 기울기 소실이 발생합니다. 또는 반대로 기울기가 폭발적으로 커지는 기울기 폭발(gradient explosion)이 발생합니다. 어느 쪽이든 모델은 제대로 학습하지 못합니다.

---

## 핵심 요약

| | 리사이징 | 정규화 |
|---|---|---|
| **해결 문제** | 입력 크기 불일치 | 픽셀값 범위와 분포 문제 |
| **방법** | 보간법으로 목표 크기로 변환 | 0~1 스케일링 또는 평균·표준편차 표준화 |
| **이유** | 모델 구조의 고정 입력 크기, 배치 처리 | 학습 안정성, 수렴 속도, 활성화 함수 포화 방지 |
| **주의점** | 비율 유지, 정보 손실 최소화 | 전이 학습 시 사전 학습 데이터셋 기준 적용 |

리사이징은 모델이 데이터를 받아들일 수 있도록 형태를 맞추는 과정이고, 정규화는 모델이 데이터를 효율적으로 학습할 수 있도록 값의 범위와 분포를 조정하는 과정입니다. 두 과정 모두 모델의 성능과 학습 안정성에 직접적인 영향을 미치는 필수 전처리 단계입니다.
