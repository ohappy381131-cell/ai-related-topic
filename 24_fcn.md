# 24. FCN(Fully Convolutional Networks) — 주요 특징과 CNN 분류 모델과의 차이

본 글에서는 Semantic Segmentation의 출발점이 된 FCN이 무엇인지, 기존 CNN 분류 모델과 어떤 구조적 차이가 있는지를 하나의 흐름 속에서 정리합니다. FC Layer를 합성곱으로 교체하는 단순한 아이디어가 어떻게 픽셀 단위 예측을 가능하게 했는지, 그리고 스킵 구조가 왜 필요한지를 구조적으로 이해할 수 있도록 설명합니다.

## 1. FCN이 등장한 배경

앞서 살펴본 것처럼 Semantic Segmentation은 이미지의 모든 픽셀에 클래스 레이블을 부여하는 태스크입니다. FCN이 등장하기 전까지 이 문제를 풀려는 시도는 대부분 비효율적이었습니다. 가장 단순한 방법은 이미지의 각 픽셀을 중심으로 패치(patch)를 잘라내 분류 모델에 넣고, 중심 픽셀의 클래스를 예측하는 방식이었습니다. 픽셀 수만큼 분류를 반복해야 하므로 계산 비용이 천문학적이었고, 패치마다 독립적으로 처리하므로 주변 맥락 정보도 제한적으로만 활용되었습니다.

FCN(Fully Convolutional Networks)은 2015년 Long, Shelhamer, Darrell이 발표한 논문에서 제안한 구조로, 이 비효율성을 근본적으로 해결했습니다. 핵심 아이디어는 단순하지만 강력합니다. **기존 분류 모델에서 완전 연결 레이어를 합성곱 레이어로 교체하면, 이미지 전체에 대한 픽셀 단위 예측을 한 번의 순전파로 수행할 수 있다**는 것입니다.

---

## 2. 기존 CNN 분류 모델의 구조

FCN과의 차이를 이해하려면 먼저 기존 CNN 분류 모델의 구조를 명확히 이해해야 합니다.

AlexNet, VGG, ResNet 같은 이미지 분류 모델은 크게 두 부분으로 구성됩니다. 전반부는 합성곱 레이어들로 이루어진 특성 추출기(feature extractor)입니다. 입력 이미지에서 층을 거치면서 점점 추상적인 특성을 추출하고, 동시에 공간 해상도가 줄어듭니다. 후반부는 완전 연결 레이어(Fully Connected Layer, FC Layer)들로 이루어진 분류기(classifier)입니다. 합성곱 레이어의 출력을 1차원 벡터로 펼친(flatten) 뒤, FC 레이어를 거쳐 최종 클래스 확률을 출력합니다.

```
기존 CNN 분류 모델:

입력 이미지 (224×224×3)
    ↓
합성곱 레이어들 (특성 추출, 해상도 감소)
    ↓
7×7×512 특성 맵
    ↓
Flatten → 25,088차원 벡터
    ↓
FC Layer (25,088 → 4,096)
    ↓
FC Layer (4,096 → 4,096)
    ↓
FC Layer (4,096 → 1,000)
    ↓
Softmax → 클래스 확률 벡터 (1,000)
```

이 구조의 결정적 한계는 두 가지입니다. 첫째, Flatten과 FC Layer가 공간 정보를 완전히 버립니다. 7×7 특성 맵을 1차원 벡터로 펼치는 순간 어느 위치에 어떤 특성이 있었는지 정보가 사라집니다. 둘째, 고정된 입력 크기만 처리할 수 있습니다. FC Layer의 파라미터 수가 입력 크기에 종속되기 때문에, 학습 때 사용한 크기(예: 224×224)와 다른 크기의 이미지를 처리할 수 없습니다.

---

## 3. FCN의 핵심 아이디어: FC Layer를 합성곱으로 교체

FCN의 핵심은 **완전 연결 레이어를 1×1 합성곱 레이어로 교체**하는 것입니다. 이 단순한 변환이 모델의 성격을 근본적으로 바꿉니다.

수학적으로 FC Layer와 1×1 합성곱은 동일한 연산을 수행합니다. 차이는 FC Layer가 공간 정보를 버리는 반면, 1×1 합성곱은 공간 정보를 유지한다는 것입니다.

```
기존 분류 모델의 FC Layer → FCN의 1×1 합성곱 교체:

FC Layer (25,088 → 4,096)
= 1×1 합성곱 (채널: 512 → 4,096, 공간: 7×7 유지)

FC Layer (4,096 → 4,096)
= 1×1 합성곱 (채널: 4,096 → 4,096, 공간: 7×7 유지)

FC Layer (4,096 → 1,000)
= 1×1 합성곱 (채널: 4,096 → 1,000, 공간: 7×7 유지)
```

FC Layer를 교체하고 나면 최종 출력이 1,000차원 벡터가 아니라 7×7×1,000 특성 맵이 됩니다. 각 7×7 위치가 해당 공간 영역의 클래스 예측값을 담고 있습니다. 이것이 FCN의 출발점입니다.

---

## 4. FCN의 주요 특징

### 임의 크기 입력 처리

FC Layer를 합성곱으로 교체하면 입력 크기 제약이 사라집니다. 합성곱 연산은 커널이 이미지 위를 슬라이딩하는 방식이므로, 입력 크기에 무관하게 적용할 수 있습니다. 224×224 이미지를 처리하면 7×7 출력 맵이, 448×448 이미지를 처리하면 14×14 출력 맵이 나옵니다. 입력 크기가 달라지면 출력 맵 크기도 비례해서 달라집니다.

### 픽셀 단위 예측 (Dense Prediction)

FCN의 출력은 공간 정보가 보존된 특성 맵입니다. 이 맵의 각 위치가 원본 이미지의 해당 영역에 대한 클래스 예측을 담고 있습니다. 이미지 전체를 한 번의 순전파로 처리해 픽셀 단위 예측을 동시에 수행합니다. 패치별 독립 처리 방식과 비교하면 계산 효율이 압도적으로 높습니다.

### 업샘플링 (Upsampling)을 통한 해상도 복원

합성곱과 풀링을 거치면서 공간 해상도가 줄어듭니다. 224×224 입력이 최종적으로 7×7 특성 맵이 되었다면, 이것을 다시 224×224 크기의 분할 맵으로 복원해야 합니다. FCN은 업샘플링(Upsampling)을 사용해 이 해상도를 복원합니다.

FCN이 사용하는 업샘플링 방법은 전치 합성곱(Transposed Convolution, 또는 Deconvolution)입니다. 일반 합성곱이 큰 입력에서 작은 출력을 만든다면, 전치 합성곱은 작은 입력에서 큰 출력을 만듭니다. 학습 가능한 파라미터를 통해 업샘플링이 이루어지므로, 단순한 보간법보다 더 세밀한 복원이 가능합니다.

---

## 5. FCN의 스킵 구조 (Skip Architecture)

단순히 최종 특성 맵을 32배 업샘플링하면(FCN-32s) 해상도 복원이 너무 거칠어 경계가 뭉개집니다. 압축 과정에서 세밀한 공간 정보가 많이 손실되기 때문입니다.

FCN 논문은 이를 해결하기 위해 스킵 구조(skip architecture)를 제안했습니다. 최종 레이어뿐만 아니라 중간 레이어의 예측도 결합해 더 세밀한 분할 맵을 만드는 방법입니다.

```
FCN-32s: 최종 특성 맵만 32배 업샘플링
→ 거친 예측, 경계 뭉개짐

FCN-16s: 최종 특성 맵 2배 업샘플링 + Pool4 출력 결합 → 16배 업샘플링
→ 더 세밀한 예측

FCN-8s: FCN-16s 결과 + Pool3 출력 결합 → 8배 업샘플링
→ 가장 세밀한 예측

스킵 연결 구조:

Pool3 ──────────────────────────────┐
Pool4 ────────────────────┐         │ + → FCN-8s 출력
최종 특성 맵 → 2× 업샘플링 → + → 2× 업샘플링 ─┘
```

초기 레이어의 특성 맵은 공간 해상도가 높고 세밀한 경계 정보를 담고 있습니다. 후기 레이어의 특성 맵은 해상도는 낮지만 의미론적 정보가 풍부합니다. 스킵 구조는 이 두 가지를 결합해 세밀한 경계와 정확한 의미 분류를 동시에 달성합니다.

---

## 6. FCN과 기존 CNN 분류 모델의 차이 종합

| | CNN 분류 모델 | FCN |
|---|---|---|
| **최종 레이어** | 완전 연결 레이어 (FC Layer) | 1×1 합성곱 레이어 |
| **출력 형태** | 클래스 확률 벡터 (1D) | 픽셀별 클래스 맵 (2D) |
| **입력 크기** | 고정 (예: 224×224) | 임의 크기 가능 |
| **공간 정보** | Flatten으로 버림 | 끝까지 보존 |
| **예측 단위** | 이미지 전체 | 픽셀 단위 |
| **업샘플링** | 없음 | 전치 합성곱으로 해상도 복원 |
| **스킵 연결** | 없음 | 중간 레이어 출력 결합 |
| **주요 활용** | 이미지 분류 | Semantic Segmentation |

---

## 7. FCN의 의의와 한계

FCN은 Semantic Segmentation 역사에서 결정적인 전환점입니다. 픽셀 단위 예측을 end-to-end 학습으로 처음 구현했고, 이후 등장하는 U-Net, DeepLab, PSPNet 등 모든 Semantic Segmentation 모델의 이론적 기반이 되었습니다. 기존 분류 모델(AlexNet, VGG 등)의 사전 학습된 가중치를 그대로 활용할 수 있어 전이 학습도 자연스럽게 적용됩니다.

그러나 한계도 있습니다. 스킵 구조를 사용해도 업샘플링 과정에서 경계가 충분히 세밀하게 복원되지 않는 경우가 있습니다. 또한 전역적 맥락(global context) 정보를 충분히 활용하지 못해 넓은 영역에 걸친 객체를 일관되게 분할하는 데 어려움이 있습니다. 이 한계를 극복하기 위해 U-Net의 체계적인 스킵 연결, DeepLab의 Atrous Convolution, PSPNet의 Pyramid Pooling 같은 개선이 이루어졌습니다.

---

## 핵심 요약

FCN의 핵심은 단 하나입니다. 기존 분류 모델에서 공간 정보를 버리는 FC Layer를 공간 정보를 보존하는 1×1 합성곱으로 교체한 것입니다. 이 단순한 변환이 고정 크기 이미지를 받아 하나의 클래스를 출력하던 모델을, 임의 크기 이미지를 받아 픽셀 단위 클래스 맵을 출력하는 모델로 바꿨습니다. 여기에 업샘플링으로 해상도를 복원하고, 스킵 구조로 세밀한 경계 정보를 보완해 Semantic Segmentation의 첫 번째 실용적인 end-to-end 모델을 완성했습니다.
