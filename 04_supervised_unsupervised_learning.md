# 지도 학습과 비지도 학습 — 레이블의 존재 여부로 나뉘는 두 가지 학습 패러다임

본 글에서는 머신러닝의 두 핵심 학습 방식인 지도 학습(Supervised Learning)과 비지도 학습(Unsupervised Learning)의 개념, 구조, 알고리즘, 선택 기준을 하나의 흐름 속에서 정리합니다. 두 패러다임이 각각 어떤 문제를 풀기 위해 설계된 것인지, 그리고 실무에서 어떤 기준으로 선택하는지를 구조적으로 이해할 수 있도록 설명합니다.

## 1. 가장 근본적인 차이: 레이블(Label)

머신러닝이 학습하는 방식은 **정답지가 있느냐 없느냐**로 갈립니다.

- **지도 학습:** 입력(X)과 정답(y)을 함께 제공 → 모델이 X→y 매핑을 학습
- **비지도 학습:** 입력(X)만 제공 → 모델이 데이터 내부 구조를 스스로 발견

---

## 2. 지도 학습 (Supervised Learning)

### 작동 원리

```
[입력 X] + [정답 y] → 모델 학습 → 새 X가 들어오면 y를 예측
```

### 학습 과정 단계별

```
1단계: 훈련 데이터(X, y) 준비
2단계: 모델에 X 입력 → 예측값 ŷ 출력
3단계: ŷ와 실제 y의 오차(loss) 계산
4단계: 오차를 줄이는 방향으로 모델 파라미터 업데이트
5단계: 수렴할 때까지 2~4 반복
6단계: 새로운 X로 예측
```

### 두 가지 하위 유형

**① 분류 (Classification)** — 출력이 범주(카테고리)

| 입력 | 정답 레이블 |
|---|---|
| 이메일 본문 | 스팸 / 정상 |
| 종양 이미지 | 악성 / 양성 |
| 고객 데이터 | 이탈 / 유지 |

주요 알고리즘: Logistic Regression, Decision Tree, Random Forest, SVM, Neural Network

**② 회귀 (Regression)** — 출력이 연속 수치

| 입력 | 예측값 |
|---|---|
| 집 면적, 위치, 연식 | 집값 (억 원) |
| 광고 지출 | 매출액 |
| 기온, 습도 | 전력 소비량 |

주요 알고리즘: Linear Regression, Ridge, Lasso, XGBoost

### 장단점

| 장점 | 단점 |
|---|---|
| 명확한 성능 지표 존재 (정확도, F1, RMSE) | 레이블 수집 비용이 높음 |
| 예측 결과 해석 용이 | 레이블 편향이 모델 편향으로 직결 |
| 실무 적용 범위 넓음 | 레이블 없는 데이터 활용 불가 |

---

## 3. 비지도 학습 (Unsupervised Learning)

### 작동 원리

```
[입력 X만] → 모델이 패턴/구조 발견 → 군집, 압축, 분포 출력
```

정답이 없으므로 "맞다/틀리다"가 없습니다. 대신 **얼마나 의미 있는 구조를 발견했는가**로 평가합니다.

### 학습 과정 단계별

```
1단계: 레이블 없는 데이터(X)만 준비
2단계: 알고리즘이 내부 구조 탐색 (거리, 밀도, 분산 등 계산)
3단계: 군집 / 저차원 표현 / 이상치 출력
4단계: 인간이 결과를 해석하고 의미 부여  ← 지도 학습과 결정적 차이
```

### 세 가지 하위 유형

**① 군집화 (Clustering)** — 유사한 것끼리 묶기

비슷한 특성을 가진 데이터 포인트를 자동으로 그룹화합니다.

주요 알고리즘:
- **K-Means:** 미리 군집 수(K)를 지정, 중심점 반복 이동
- **DBSCAN:** 밀도 기반, 이상치(outlier) 자동 탐지
- **Hierarchical Clustering:** 계층적 트리 구조로 군집화

**② 차원 축소 (Dimensionality Reduction)** — 고차원 → 저차원 압축

수백~수천 개의 변수를 2~3개의 핵심 축으로 압축합니다.

주요 알고리즘:
- **PCA (주성분 분석):** 분산이 최대인 방향으로 축 재설정
- **t-SNE:** 고차원 유사도를 저차원에서 보존, 시각화 특화
- **UMAP:** t-SNE보다 빠르고 전역 구조 보존

**③ 이상치 탐지 (Anomaly Detection)** — 정상 패턴에서 벗어난 것 찾기

주요 알고리즘: Autoencoder, Isolation Forest, One-Class SVM

### 장단점

| 장점 | 단점 |
|---|---|
| 레이블 불필요 → 대량 데이터 바로 활용 | 결과 평가 기준 불명확 |
| 사전에 몰랐던 패턴 발견 가능 | 발견한 군집의 의미를 인간이 해석해야 함 |
| 탐색적 데이터 분석(EDA)에 강함 | 예측 정확도 수치로 비교 어려움 |

---

## 4. 두 패러다임 비교

| 항목 | 지도 학습 | 비지도 학습 |
|---|---|---|
| **레이블** | 필수 | 없음 |
| **목표** | 예측 | 구조 발견 |
| **성능 평가** | 정확도, F1, RMSE 등 수치 | 실루엣 계수, 시각적 해석 |
| **인간 역할** | 레이블 제공 | 결과 해석 |
| **데이터 요구량** | 레이블된 데이터 | 양이 많을수록 유리 |
| **대표 과제** | 스팸 분류, 가격 예측 | 고객 세분화, 이상치 탐지 |

---

## 5. 그 외 주요 패러다임

| 패러다임 | 특징 |
|---|---|
| **준지도 학습** (Semi-supervised) | 소량 레이블 + 대량 비레이블 혼합 사용 |
| **자기지도 학습** (Self-supervised) | 데이터 자체에서 레이블 자동 생성 (GPT, BERT 방식) |
| **강화 학습** (Reinforcement Learning) | 보상/페널티 신호로 행동 최적화, 정답 없이 환경과 상호작용 |

---

## 6. 선택 기준 가이드

```
데이터에 레이블이 있는가?
├── YES → 지도 학습
│         출력이 범주형? → 분류 (Classification)
│         출력이 연속형? → 회귀 (Regression)
│
└── NO  → 목적이 무엇인가?
          패턴 발견/군집    → 군집화 (K-Means, DBSCAN)
          구조 압축/시각화  → 차원 축소 (PCA, UMAP)
          이상치 탐지       → 밀도 추정 (Autoencoder)
```

> **지도 학습 =** 인간이 정의한 정답을 향해 수렴하는 학습  
> **비지도 학습 =** 데이터가 스스로 드러내는 구조를 포착하는 학습
