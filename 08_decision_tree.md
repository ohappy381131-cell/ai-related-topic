# 결정 트리(Decision Tree) — 장점과 단점

본 글에서는 결정 트리의 구조와 학습 원리를 바탕으로, 이 알고리즘이 가진 강점과 한계를 하나의 흐름 속에서 정리합니다. 결정 트리가 단독 모델로서 어떤 상황에 적합하고, 왜 현대 머신러닝에서 앙상블의 기반 모델로 주로 활용되는지를 구조적으로 이해할 수 있도록 설명합니다.

## 1. 결정 트리란

결정 트리(Decision Tree)는 데이터를 특정 조건에 따라 반복적으로 분기(split)하면서 예측을 만들어내는 모델입니다. 이름 그대로 나무를 거꾸로 세운 것 같은 구조를 가집니다. 맨 위의 루트 노드(root node)에서 시작해, 각 노드에서 "이 조건을 만족하는가?"를 질문하며 데이터를 두 그룹으로 나눕니다. 이 과정을 리프 노드(leaf node), 즉 더 이상 분기하지 않는 끝 노드에 도달할 때까지 반복합니다. 리프 노드에 도달하면 그 노드에 해당하는 예측값을 출력합니다.

```
                [나이 < 30?]
               /            \
           YES               NO
      [연봉 < 3000?]     [신용점수 < 600?]
       /        \            /         \
     YES         NO        YES          NO
   [거절]     [승인]     [거절]       [승인]
```

분류 문제에서는 각 리프 노드가 클래스 레이블을, 회귀 문제에서는 수치값을 출력합니다. 구조 자체가 인간의 의사결정 방식과 매우 유사해서 직관적으로 이해하기 쉽다는 것이 결정 트리의 가장 큰 특징입니다.

---

## 2. 학습 과정: 어떻게 분기 기준을 정하는가

결정 트리가 학습한다는 것은 각 노드에서 데이터를 가장 잘 분리하는 특성(feature)과 임계값(threshold)을 찾는 과정입니다. 이때 "가장 잘 분리한다"는 기준을 불순도(impurity)로 측정합니다. 분기 후 각 그룹이 하나의 클래스로 최대한 순수하게 구성될수록 좋은 분기입니다.

불순도를 측정하는 대표적인 지표로 두 가지가 있습니다. 지니 불순도(Gini Impurity)는 한 노드에서 무작위로 샘플을 골랐을 때 잘못 분류할 확률을 나타냅니다. 정보 이득(Information Gain)은 엔트로피(entropy) 개념을 사용해 분기 전후의 불확실성 감소량을 측정합니다. 두 지표 모두 분기 후 각 그룹이 순수할수록 값이 낮아지며, 알고리즘은 이 값을 최소화하는 분기 조건을 매 노드마다 탐색합니다.

---

## 3. 장점

### 해석 가능성이 높다

결정 트리의 가장 두드러진 강점은 모델의 예측 과정을 사람이 직접 따라갈 수 있다는 점입니다. "나이가 30세 미만이고 연봉이 3,000만 원 이하이기 때문에 대출이 거절되었다"처럼 예측의 이유를 명확하게 설명할 수 있습니다. 이를 모델의 해석 가능성(interpretability) 또는 설명 가능성(explainability)이라고 합니다. 금융, 의료, 법률처럼 의사결정의 근거를 반드시 설명해야 하는 도메인에서 이 특성은 매우 중요합니다.

### 데이터 전처리 부담이 적다

결정 트리는 특성 스케일링(feature scaling)이 필요하지 않습니다. 선형 회귀나 SVM 같은 모델은 변수들의 단위가 다르면 스케일을 맞춰야 하지만, 결정 트리는 각 노드에서 단순히 "이 값이 임계값보다 큰가 작은가"만 판단하므로 변수 간 스케일 차이가 결과에 영향을 주지 않습니다. 또한 결측값(missing value) 처리나 이상치(outlier) 제거 없이도 어느 정도 작동합니다.

### 수치형과 범주형 데이터를 모두 다룰 수 있다

많은 모델이 범주형 변수를 수치로 변환하는 인코딩 작업을 요구하지만, 결정 트리는 수치형과 범주형 특성을 큰 추가 처리 없이 모두 처리할 수 있습니다. 다양한 형태의 데이터가 혼재하는 실무 환경에서 유리합니다.

### 비선형 관계를 자연스럽게 포착한다

결정 트리는 데이터가 선형 관계를 따른다는 가정이 없습니다. 분기를 반복하면서 복잡한 비선형 경계도 표현할 수 있습니다. 선형 모델이 포착하지 못하는 패턴도 트리 구조로는 자연스럽게 학습할 수 있습니다.

### 특성 중요도를 제공한다

학습이 끝난 후 어떤 특성이 분기에 많이 사용되었는지를 기반으로 특성 중요도(feature importance)를 계산할 수 있습니다. 수백 개의 변수 중 어떤 변수가 예측에 핵심적인 역할을 하는지를 파악하는 탐색적 분석에 유용합니다.

---

## 4. 단점

### 과적합에 매우 취약하다

결정 트리의 가장 심각한 단점입니다. 트리의 깊이를 제한하지 않으면 훈련 데이터의 모든 패턴, 심지어 노이즈까지 학습해버립니다. 극단적으로는 훈련 데이터의 모든 샘플을 완벽하게 분류하는 트리가 만들어지는데, 이 트리는 새로운 데이터에 대해 성능이 대폭 낮아집니다. 편향-분산 트레이드오프에서 결정 트리는 고분산(high variance) 모델의 전형적인 예입니다.

이를 방지하기 위해 트리의 최대 깊이(max depth), 리프 노드의 최소 샘플 수(min samples per leaf), 분기 시 최소 샘플 수(min samples to split) 같은 하이퍼파라미터로 트리 성장을 제한하는 가지치기(pruning)가 필요합니다.

### 데이터 변화에 불안정하다

결정 트리는 분산(variance)이 높습니다. 훈련 데이터가 조금만 바뀌어도 트리의 구조가 크게 달라질 수 있습니다. 루트 노드에서의 첫 번째 분기 조건이 바뀌면 그 아래의 모든 구조가 연쇄적으로 달라집니다. 이 불안정성 때문에 단일 결정 트리만으로는 신뢰할 수 있는 예측을 보장하기 어렵습니다.

### 클래스 불균형에 취약하다

훈련 데이터에 특정 클래스가 압도적으로 많으면 결정 트리는 다수 클래스 쪽으로 편향된 분기를 만드는 경향이 있습니다. 소수 클래스(minority class)를 제대로 학습하지 못하고 무시하는 방향으로 트리가 성장할 수 있습니다. 이 경우 클래스 가중치를 조정하거나 언더샘플링·오버샘플링 기법을 적용해야 합니다.

### 직선형 경계만 표현 가능하다

결정 트리의 분기는 항상 하나의 특성에 대한 수직/수평 조건입니다. 실제 데이터의 결정 경계가 대각선이나 곡선 형태라면, 결정 트리는 이를 계단 모양의 직선 경계로 근사합니다. 분기를 많이 늘리면 어느 정도 근사할 수 있지만, 이는 곧 과적합으로 이어집니다.

---

## 5. 단점을 극복하는 방법: 앙상블

결정 트리의 단점, 특히 과적합과 불안정성은 단일 트리를 여러 개 결합하는 앙상블(Ensemble) 방법으로 대부분 해결됩니다. 결정 트리는 단독으로 쓰이기보다 더 강력한 앙상블 모델의 기반 모델(base model)로 활용될 때 진가를 발휘합니다.

Random Forest는 수백 개의 결정 트리를 서로 다른 데이터 샘플과 특성 조합으로 각각 학습시킨 뒤, 그 예측을 평균(회귀) 또는 투표(분류)로 결합합니다. 개별 트리의 고분산이 평균 과정에서 상쇄되어 훨씬 안정적인 예측이 가능해집니다.

XGBoost, LightGBM 같은 Boosting 계열은 이전 트리가 틀린 데이터에 다음 트리가 집중하는 방식으로 순차적으로 학습해 편향을 줄입니다. 현재 정형 데이터(tabular data) 분야에서 가장 강력한 성능을 보이는 알고리즘들이 모두 결정 트리 기반의 Boosting입니다.

## 핵심 요약

| | 내용 |
|---|---|
| **핵심 구조** | 조건 분기를 반복해 데이터를 분류/예측하는 트리 구조 |
| **주요 장점** | 해석 가능성, 전처리 부담 적음, 비선형 관계 표현, 특성 중요도 제공 |
| **주요 단점** | 과적합 취약, 데이터 변화에 불안정, 클래스 불균형 취약 |
| **실무 활용** | 단독보다 Random Forest, XGBoost 같은 앙상블의 기반 모델로 주로 사용 |

결정 트리는 그 자체로 완성된 모델이기도 하지만, 현대 머신러닝에서는 강력한 앙상블 모델들의 출발점으로서의 역할이 더 큽니다. 결정 트리의 구조와 한계를 정확히 이해하는 것이 Random Forest와 Boosting 계열 알고리즘을 깊이 이해하는 전제가 됩니다.
