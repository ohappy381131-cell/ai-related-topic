# 4. K-폴드 교차 검증(K-Fold Cross Validation) — K값 선택 시 고려해야 할 점

본 글에서는 K-폴드 교차 검증의 개념과 K값이 모델 평가에 미치는 영향을 하나의 흐름 속에서 정리합니다. K값이 단순한 반복 횟수가 아니라 편향, 분산, 계산 비용 사이의 균형을 결정하는 설계 선택임을 구조적으로 이해할 수 있도록 설명합니다.

---

## 목차

1. K-폴드 교차 검증이란
2. K값이 왜 중요한가
3. K가 클 때: 저편향, 고분산, 고비용
4. K가 작을 때: 고편향, 저분산, 저비용
5. 극단적 경우: LOOCV
6. 데이터 크기와 K의 관계
7. 왜 K=5 또는 K=10이 관행인가
8. 클래스 불균형과 Stratified K-Fold
9. 시계열 데이터에서의 주의점

---

## 1. K-폴드 교차 검증이란

모델을 평가할 때 데이터를 단순히 훈련/테스트로 한 번만 나누면 문제가 생깁니다. 어떻게 나누느냐에 따라 성능이 달라질 수 있고, 운 좋게 쉬운 테스트 데이터가 걸리면 모델이 실제보다 좋아 보이거나, 반대로 나쁘게 보일 수 있습니다. K-폴드 교차 검증은 이 불안정성을 해결하기 위해 설계된 방법입니다.

```
전체 데이터를 K개의 폴드(fold)로 균등하게 나눈다
→ K번 반복한다
   매 반복마다 폴드 하나를 검증 데이터로 쓰고
   나머지 K-1개 폴드로 훈련한다
→ K개의 검증 성능을 평균 내어 최종 성능으로 사용한다
```

예를 들어 K=5라면 데이터를 5등분하고, 각 폴드가 한 번씩 검증 역할을 맡아 총 5번 학습과 평가를 수행합니다. 모든 데이터가 적어도 한 번은 검증에 사용되므로, 단일 분할보다 훨씬 안정적인 성능 추정이 가능합니다.

---

## 2. K값이 왜 중요한가

K는 단순한 반복 횟수가 아닙니다. K값이 결정되면 매 반복마다 훈련에 사용되는 데이터의 양과 검증에 사용되는 데이터의 양이 동시에 결정됩니다.

K가 크면 훈련 데이터가 많아지고 검증 데이터가 줄어듭니다. K가 작으면 반대입니다. 이 비율의 변화가 편향(bias)과 분산(variance) 모두에 영향을 미치며, 계산 비용까지 결정합니다. K값은 이 세 가지 요소 사이의 균형을 조율하는 선택입니다.

---

## 3. K가 클 때: 저편향, 고분산, 고비용

K를 크게 설정하면, 예를 들어 K=10이라면, 매 반복마다 전체 데이터의 90%로 훈련하고 10%로 검증합니다. 훈련 데이터가 많으므로 모델이 실제 분포를 잘 학습할 수 있고, 성능 추정의 편향(bias)이 낮아집니다. 즉, 추정된 성능이 실제 성능에 가깝다는 뜻입니다.

그러나 검증 데이터가 적어지면 각 폴드의 검증 성능이 그 폴드에 포함된 특정 데이터 포인트들에 민감해집니다. 결과적으로 K개의 검증 점수 사이의 변동성, 즉 분산(variance)이 커집니다. 또한 K번의 학습을 반복해야 하므로 계산 비용도 늘어납니다.

---

## 4. K가 작을 때: 고편향, 저분산, 저비용

반대로 K=3처럼 작게 설정하면, 매 반복마다 전체 데이터의 약 67%로 훈련하고 33%로 검증합니다. 검증 데이터가 많아지므로 각 폴드의 검증 점수가 상대적으로 안정적이고, K개의 점수 사이 분산은 낮아집니다.

그러나 훈련 데이터가 줄어들면 모델이 전체 데이터 분포를 충분히 학습하지 못합니다. 이로 인해 실제 데이터 전체로 학습했을 때의 성능보다 낮게 추정되는 경향이 생깁니다. 이것이 성능 추정의 편향(bias)입니다. K가 작을수록 이 편향이 커집니다.

---

## 5. 극단적 경우: LOOCV

K를 극단까지 높이면 K=N, 즉 데이터 포인트 수만큼 폴드를 만드는 경우가 됩니다. 이를 Leave-One-Out Cross Validation, 줄여서 LOOCV라고 합니다.

매 반복마다 데이터 포인트 하나만을 검증에 사용하고 나머지 N-1개로 훈련합니다. 훈련 데이터가 최대이므로 편향이 가장 낮습니다. 그러나 N번의 학습을 반복해야 하므로 데이터가 많을 때 계산 비용이 폭발적으로 증가합니다. 또한 각 검증이 데이터 포인트 하나에만 의존하므로 분산이 매우 높아집니다. 일반적으로 데이터가 아주 적을 때만 실용적입니다.

---

## 6. 데이터 크기와 K의 관계

K값 선택에서 가장 직접적으로 고려해야 할 요소는 데이터의 크기입니다.

데이터가 적을 때는 K를 크게 설정하는 것이 유리합니다. 훈련 데이터를 최대한 확보해야 모델이 제대로 학습할 수 있기 때문입니다. 데이터가 수백 건 수준이라면 K=10 이상도 고려합니다.

데이터가 충분히 많을 때는 K를 굳이 크게 설정할 필요가 없습니다. K=5 정도로도 안정적인 성능 추정이 가능하고, 계산 비용을 절약할 수 있습니다. 데이터가 수십만 건 이상이라면 K=3도 충분히 신뢰할 만한 추정을 제공합니다.

| 데이터 규모 | 권장 K값 | 이유 |
|---|---|---|
| 소규모 (수백 건 이하) | K=10 이상, 또는 LOOCV | 훈련 데이터 최대 확보 |
| 중간 (수천~수만 건) | K=5 또는 K=10 | 편향·분산·비용 균형 |
| 대규모 (수십만 건 이상) | K=3~5 | 계산 비용 절감, 충분한 검증 데이터 |

---

## 7. 왜 K=5 또는 K=10이 관행인가

실무에서 K=5와 K=10이 사실상 표준처럼 사용되는 데는 이유가 있습니다. 이 두 값은 편향, 분산, 계산 비용의 세 가지 요소 사이에서 경험적으로 균형이 좋다는 것이 수십 년의 연구와 실무를 통해 검증되어 왔기 때문입니다. 특히 데이터 크기나 문제 유형에 대한 사전 정보가 없을 때의 합리적인 출발점입니다.

---

## 8. 클래스 불균형과 Stratified K-Fold

분류 문제에서 클래스 불균형이 있을 때는 일반 K-폴드를 그대로 사용하면 문제가 생길 수 있습니다. 무작위로 나누다 보면 특정 폴드에 소수 클래스(minority class)가 거의 없거나 아예 없는 경우가 발생합니다. 이런 폴드로 검증하면 성능 추정이 왜곡됩니다.

이를 해결하는 것이 계층적 K-폴드(Stratified K-Fold)입니다. 각 폴드에 원본 데이터와 동일한 클래스 비율이 유지되도록 나누는 방법입니다. 분류 문제라면 일반 K-폴드 대신 Stratified K-Fold를 기본으로 사용하는 것이 권장됩니다.

---

## 9. 시계열 데이터에서의 주의점

시계열 데이터(time series data)에는 일반 K-폴드를 적용해서는 안 됩니다. 시계열 데이터는 순서가 중요합니다. 미래의 데이터로 과거를 예측하는 훈련은 현실에서 불가능한 상황을 만들어내며, 이렇게 학습된 모델은 실제로 배포했을 때 성능이 대폭 낮아집니다. 이를 데이터 누수(data leakage)라고 합니다.

시계열 데이터에는 시간 순서를 유지하는 Time Series Split 방식을 사용합니다. 항상 과거 데이터로 훈련하고 미래 데이터로 검증하는 방향을 지킵니다.

---

## 핵심 요약

| 고려 요소 | K를 크게 | K를 작게 |
|---|---|---|
| **편향** | 낮음 (훈련 데이터 많음) | 높음 (훈련 데이터 적음) |
| **분산** | 높음 (검증 데이터 적음) | 낮음 (검증 데이터 많음) |
| **계산 비용** | 높음 | 낮음 |
| **적합한 상황** | 데이터 적을 때 | 데이터 많을 때 |

K값 선택에는 절대적인 정답이 없습니다. 데이터의 크기, 클래스 불균형 여부, 데이터의 순서 의존성, 허용 가능한 계산 비용을 함께 고려해야 합니다. 대부분의 상황에서는 K=5 또는 K=10으로 시작하고, 데이터 특성에 따라 조정하는 것이 실용적인 접근입니다.
