## 1. 세 가지 비전 태스크의 스펙트럼

컴퓨터 비전에서 이미지를 이해하는 방식은 얼마나 세밀하게 분석하느냐에 따라 스펙트럼이 있습니다. 이미지 분류, 객체 탐지, Semantic Segmentation은 이 스펙트럼의 세 지점을 대표합니다. Semantic Segmentation을 제대로 이해하려면 이 세 가지를 함께 놓고 비교하는 것이 가장 명확합니다.

```
이미지 분류          객체 탐지              Semantic Segmentation
(Classification)    (Object Detection)
┌──────────┐        ┌──────────┐           ┌──────────┐
│          │        │ ┌──────┐ │           │▒▒▒▒▒▒▒▒▒▒│
│  고양이   │        │ │고양이│ │           │▒▒고양이▒▒│
│  사진    │        │ └──────┘ │           │░░░░░░░░░░│
│          │        │          │           │░░배경░░░░│
└──────────┘        └──────────┘           └──────────┘
"고양이"            "고양이, 여기에 있음"    "각 픽셀이 무엇인가"
이미지 전체         바운딩 박스 단위         픽셀 단위
```

이미지 분류는 이미지 전체에 하나의 레이블을 붙입니다. 객체 탐지는 객체의 위치를 바운딩 박스로 표시하고 클래스를 붙입니다. Semantic Segmentation은 이미지의 모든 픽셀에 클래스 레이블을 붙입니다. 분석의 단위가 이미지 → 박스 → 픽셀로 점점 세밀해집니다.

---

## 2. Semantic Segmentation이란

Semantic Segmentation(의미론적 분할)은 이미지의 **모든 픽셀**이 어떤 클래스에 속하는지를 예측하는 태스크입니다. 입력 이미지와 동일한 크기의 출력 맵을 생성하되, 각 위치의 값이 해당 픽셀의 클래스 레이블입니다.

예를 들어 도로 주행 이미지라면, 모든 픽셀을 도로, 하늘, 건물, 보행자, 자동차, 나무 중 하나로 분류합니다. 출력은 각 픽셀의 클래스가 색상으로 표현된 분할 맵(segmentation map)입니다.

```
입력 이미지 (H × W × 3)
    ↓
Semantic Segmentation 모델
    ↓
분할 맵 (H × W × C)
→ 각 픽셀에 C개 클래스 중 하나의 레이블 부여
```

"Semantic"이라는 단어가 붙은 이유는 픽셀을 단순히 색상이나 질감으로 구분하는 것이 아니라, 의미론적 범주(고양이, 하늘, 도로 등)로 분류하기 때문입니다.

---

## 3. 이미지 분류와의 차이

### 예측 단위의 차이

가장 근본적인 차이는 예측의 단위입니다. 이미지 분류는 이미지 전체를 하나의 단위로 보고 단 하나의 레이블을 예측합니다. Semantic Segmentation은 이미지를 구성하는 수백만 개의 픽셀 각각에 레이블을 예측합니다. 분류 문제 하나를 픽셀 수만큼 반복하는 것과 개념적으로 유사합니다.

### 출력의 형태 차이

이미지 분류의 출력은 클래스 확률 벡터 하나입니다. 클래스가 10개라면 길이 10인 벡터입니다. Semantic Segmentation의 출력은 입력 이미지와 같은 크기의 2차원 맵입니다. 224×224 이미지라면 224×224 크기의 레이블 맵을 출력합니다.

### 공간 정보의 활용 차이

이미지 분류 모델은 공간 정보를 점진적으로 압축합니다. CNN이 층을 거치면서 특성 맵의 크기가 줄어들고, 마지막에 완전 연결 레이어가 공간 정보를 완전히 버리고 클래스 확률만 출력합니다. 이 과정에서 "어디에 있는가"라는 정보는 사라집니다.

Semantic Segmentation은 이 공간 정보를 끝까지 보존해야 합니다. 픽셀 단위로 분류해야 하므로 각 픽셀의 위치 정보가 출력까지 유지되어야 합니다. 이것이 Semantic Segmentation 모델 구조가 분류 모델과 근본적으로 다른 이유입니다.

---

## 4. Semantic Segmentation의 핵심 구조: Encoder-Decoder

Semantic Segmentation 모델은 대부분 인코더-디코더(Encoder-Decoder) 구조를 사용합니다. 이 구조는 두 가지 상반된 요구를 동시에 만족시키기 위해 설계되었습니다.

인코더는 특성을 추출하기 위해 공간 해상도를 줄입니다. 풀링과 스트라이드 합성곱으로 특성 맵을 압축하면서 의미 있는 특성을 추출합니다. 디코더는 압축된 특성 맵을 원본 이미지 크기로 복원합니다. 업샘플링(upsampling) 또는 전치 합성곱(transposed convolution)을 사용해 해상도를 점진적으로 회복합니다.

```
입력 이미지 (224×224)
    ↓ 인코더 (특성 추출, 해상도 감소)
[112×112] → [56×56] → [28×28] → [14×14] → [7×7]
    ↓ 디코더 (해상도 복원)
[14×14] → [28×28] → [56×56] → [112×112] → [224×224]
    ↓
분할 맵 (224×224)
```

### Skip Connection

인코더에서 디코더로 정보가 넘어갈 때, 압축 과정에서 세밀한 공간 정보가 손실됩니다. 이를 보완하기 위해 인코더의 중간 레이어 출력을 디코더의 대응하는 레이어에 직접 연결하는 스킵 연결(skip connection)을 사용합니다. 전반적인 의미 정보는 디코더를 통해, 세밀한 경계 정보는 스킵 연결을 통해 전달됩니다. 이 구조가 FCN과 U-Net의 핵심 아이디어입니다.

---

## 5. 대표적인 Semantic Segmentation 모델

### FCN (Fully Convolutional Network, 2015)

최초로 완전 합성곱 신경망(Fully Convolutional Network) 구조를 Semantic Segmentation에 적용한 모델입니다. 완전 연결 레이어를 모두 합성곱 레이어로 대체해 임의의 크기 이미지에 적용할 수 있으며, 픽셀 단위 예측을 end-to-end로 학습할 수 있게 했습니다.

### U-Net (2015)

의료 영상 분할을 위해 설계된 모델로, 인코더-디코더 구조에 스킵 연결을 체계적으로 결합한 것이 특징입니다. 적은 데이터로도 우수한 성능을 보여 의료 이미지 분야의 표준 모델이 되었습니다. 현재는 의료 영상을 넘어 위성 이미지, 자율주행, 산업 검사 등 다양한 분야에서 널리 활용됩니다.

### DeepLab 시리즈 (2014~2018)

Google이 개발한 모델로, Atrous Convolution(팽창 합성곱)을 도입해 해상도 손실 없이 수용 영역(receptive field)을 넓히는 방법을 제안했습니다. DeepLabv3+는 인코더-디코더 구조에 Atrous Spatial Pyramid Pooling(ASPP)을 결합해 다양한 스케일의 객체를 효과적으로 분할합니다.

---

## 6. Semantic Segmentation vs Instance Segmentation

Semantic Segmentation과 자주 혼동되는 개념이 Instance Segmentation입니다. 두 가지의 차이는 같은 클래스의 서로 다른 객체를 구분하는가에 있습니다.

```
원본 이미지: 고양이 두 마리가 있는 사진

Semantic Segmentation:
→ 두 고양이 모두 "고양이" 레이블
→ 어느 픽셀이 어떤 고양이인지 구분하지 않음

Instance Segmentation:
→ "고양이 #1", "고양이 #2"로 개별 구분
→ 같은 클래스라도 서로 다른 객체로 처리
```

Semantic Segmentation은 클래스 수준의 분할이고, Instance Segmentation은 객체 수준의 분할입니다. Mask R-CNN이 Instance Segmentation의 대표적인 모델입니다.

---

## 7. 주요 활용 분야

Semantic Segmentation은 픽셀 단위의 정밀한 이해가 필요한 곳에서 활용됩니다.

자율주행에서는 카메라 영상의 모든 픽셀을 도로, 차선, 보행자, 차량, 신호등으로 분류해 주행 경로를 판단합니다. 의료 영상 분석에서는 CT, MRI 이미지에서 종양, 장기, 병변 영역을 픽셀 단위로 분할해 진단을 보조합니다. 위성·항공 이미지 분석에서는 건물, 도로, 논밭, 산림, 수계를 픽셀 단위로 분류해 토지 이용 현황을 파악합니다. 증강 현실에서는 배경과 전경을 픽셀 단위로 분리해 실제 환경에 가상 객체를 자연스럽게 합성합니다.

## 핵심 요약

| | 이미지 분류 | 객체 탐지 | Semantic Segmentation |
|---|---|---|---|
| **예측 단위** | 이미지 전체 | 바운딩 박스 | 픽셀 |
| **출력 형태** | 클래스 벡터 | 박스 좌표 + 클래스 | 픽셀별 레이블 맵 |
| **위치 정보** | 없음 | 대략적 (박스) | 정밀 (픽셀) |
| **공간 정보** | 압축 후 버림 | 부분 보존 | 끝까지 보존 |
| **대표 모델** | ResNet, EfficientNet | YOLO, Faster R-CNN | U-Net, DeepLab |
| **주요 활용** | 이미지 태깅, 검색 | 자율주행, 보안 | 의료 영상, 자율주행 |

Semantic Segmentation은 이미지를 가장 세밀하게 이해하는 방법입니다. 이미지 분류가 "무엇이 있는가"를 묻는다면, Semantic Segmentation은 "이미지의 모든 곳이 무엇인가"를 묻습니다. 픽셀 단위의 정밀한 공간 이해가 필요한 문제에서 Semantic Segmentation은 대체 불가능한 접근입니다.
